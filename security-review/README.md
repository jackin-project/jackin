## Findings considered and deferred (not yet planned — recorded so they aren't re-audited)

### Security

- **SEC-unify-redactors** — three inconsistent secret-key predicates exist (`jackin-diagnostics::redact` regex, `secret_scrub::is_secret_key` substring, `shell_runner::is_sensitive_arg_key`); they disagree (e.g. none catches `GROK_DEPLOYMENT_KEY`), and the weakest guards the OTLP sink. Unify onto the `is_credential_env_key` predicate 005 introduces. M. Do after 002 + 005.
- **SEC-emit-redaction** — `emit_debug_line`/`emit_compact_line`/`emit_operator_notice` (`jackin-diagnostics/src/logging.rs:177-231`) and `shell_runner do_capture` (`:503-511`) do not redact; safe only by unenforced `capture_secret` caller discipline. Pipe them through `redact_text` or a type marker. S.
- **SEC-socket-nonce** — the capsule attach/control socket is authed only by uid + `0600` (`jackin-capsule/src/socket.rs:108-117`); a same-uid in-container process can inject into the operator's PTY / pop credential prompts. Add a per-launch connection nonce the container uid can't read. M, host↔capsule wire change. Partly an inherent container-local-IPC tradeoff already acknowledged in-code.

### Tests

- **TEST-firewall-enforce** — `firewall/tests.rs` covers only allowlist parsing, never that egress to a non-allowlisted address is dropped. At minimum assert the ipset stream reaches the restore command via a fake runner; ideally a privileged netns E2E. M.

### Tech debt / architecture

- **DEBT-command-seam** — 19 files call `Command::new("gh"|"op"|"docker"|"git")` directly, bypassing the `CommandRunner` port's `capture_secret` redaction; the port has no timeout/retry (only `op_cli::run_op_with_timeout` does). Extend `RunOptions` with timeout/retry/secret; migrate sites family-by-family. L. High value (security + hang-resistance); `SharedCommandRunner` (currently `#[expect(dead_code)]`) is its not-yet-wired half.
- **DEBT-auth-path-contract** — agent auth/config-dir path table duplicated across `jackin-capsule/runtime_setup.rs:25-105` (container side) and `jackin-instance/auth.rs` + `lib.rs` (host side); drift = silent auth-fallback-to-login. Extract one `AgentAuthLayout` per agent into `jackin-core`. M.
- **DEBT-github-url** — GitHub URL normalization/credential-stripping siloed in the `jackin-console` leaf (`mount_info.rs`, scp-parse written twice) plus hand-rolled copies in config/capsule. Lift `GithubRemote::parse` into a low crate. S-M.

### Dependencies

- **DEP-advisory-sync** — `.cargo/audit.toml` and `deny.toml` duplicate the advisory-ignore list with no cross-reference; add a “keep in sync” comment to both. S.

## Findings considered and rejected (do not re-audit)

- **Redaction/migration/auth coverage “gaps”** — the config/manifest migration corpus, `jackin-instance` auth (102 tests, all forward modes), cleanup/GC (44 tests), and redaction policy tests already exist and are strong. Not findings.
- **cargo-vet / cargo-crev not adopted** — documented solo-maintainer decision with a named revisit trigger in `crates/AGENTS.md`. By design.
- **Workflow hygiene** — no `pull_request_target`, actions SHA-pinned, untrusted context flows through quoted `env:` indirection. No workflow security finding raised (zizmor/gitleaks adoption is a separate planned Phase 1 item, not a current defect).

## Direction findings (maintainer's call — options, not defects)

1. **Protect the agent/repo/credentials from untrusted content (MCP governance + prompt-injection boundary).** jackin❯'s isolation protects the host from the agent, not the agent (and the mounted repo + forwarded creds) from malicious content; egress is allow-listed, so a prompt-injected agent can still exfil to an allowed destination, and nothing inspects the MCP protocol. Start with an MCP-call audit trail + human approval for write/execute tools. L, more speculative.
