## Why this matters

`cargo audit` and `cargo deny check advisories` both currently FAIL on
**RUSTSEC-2026-0204** (invalid pointer dereference in the `fmt::Pointer` impl
for `crossbeam-epoch`'s `Atomic`/`Shared`, advisory dated 2026-07-06, fixed in
`>=0.9.20`). The workspace pins `crossbeam-epoch 0.9.18` transitively (pulled at
runtime via `turso_core → crossbeam-deque → crossbeam-epoch`, and dev-only via
`criterion → rayon → crossbeam-epoch`). `crates/AGENTS.md` documents `cargo
audit` as a "deliberately duplicated RustSec PR gate," so the next
`Cargo.lock`-touching PR and the scheduled hygiene workflow go red until this is
bumped. Runtime exploitability is low (requires `Debug`/`Pointer`-formatting an
already-invalid atomic), so the concrete cost is a blocked CI gate. The fix is a
single lockfile bump with no first-party code change.

## Current state

- `Cargo.lock` — `crossbeam-epoch` is resolved at `0.9.18`. `crossbeam-epoch`
  is a transitive dependency; it does **not** appear in any `Cargo.toml`
  `[dependencies]` table, so there is no version constraint pinning it — a
  precise `cargo update` will move it freely.
- `deny.toml` — has an `[advisories]` section with an `ignore = [...]` list
  (currently `RUSTSEC-2023-0071`, `RUSTSEC-2026-0173`). RUSTSEC-2026-0204 is
  **not** ignored, and must not be added to the ignore list — the correct fix
  is to bump, not to suppress.
- `.cargo/audit.toml` — mirrors the same two advisory ignores. Do not touch
  unless the bump proves impossible (see STOP conditions).

## Steps

### Step 1: Bump the transitive dependency

Run `cargo update -p crossbeam-epoch --precise 0.9.20`. If `0.9.20` is yanked or
unavailable, retry with the next patch (`0.9.21`, etc.) — any version `>=0.9.20`
clears the advisory.

**Verify**: `git diff Cargo.lock | grep -A1 'name = "crossbeam-epoch"'` shows
`version = "0.9.20"` (or higher). No other crate's version line changed beyond
what a crossbeam-epoch bump pulls in (crossbeam-epoch may itself nudge
`crossbeam-utils`; that is fine).

### Step 2: Confirm the advisory is cleared

Run `cargo audit` and `cargo deny check advisories`.

**Verify**: neither output contains `RUSTSEC-2026-0204`. `cargo audit` prints
`0 vulnerabilities found` (or lists only advisories already in the ignore list,
not this one). `cargo deny check advisories` prints `advisories ok`.

### Step 3: Confirm no supply-chain regression

Run `cargo deny check bans licenses sources` and `cargo check --workspace --locked`.

**Verify**: `bans` stays `ok` — if the bump introduced a *new* duplicate
version of crossbeam-epoch (both 0.9.18 and 0.9.20 in the tree), `cargo deny
check bans` will flag it; in that case run `cargo update` without `--precise` to
let both consumers converge, then re-check. `cargo check` exits 0.
