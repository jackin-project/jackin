## Findings considered and deferred (not yet planned — recorded so they aren't re-audited)

High-value findings that are real but were not turned into first-wave plans (larger blast radius, a design decision, or lower leverage).

### Correctness / bugs

- **BUG-color-hex-slice** — `host_colors.rs:161-166` (duplicated in `jackin-tui` and `jackin-runtime`) slices a `#RRGGBB` OSC reply by byte under a byte-length guard; a 6-byte multibyte string panics. Guard with `is_ascii()` or parse from `as_bytes()`; dedupe the two copies. S. Low reachability but a concrete `string_slice` instance.
- **BUG-pty-unbounded-channel** — capsule `event_tx/event_rx` (`daemon.rs:202-203`) is unbounded; a runaway agent (`yes`) has no PTY backpressure and can inflate capsule RSS. Investigate depth under a flood soak before switching to a bounded/coalescing channel. LOW-MED, investigate-first.

### Performance

- **PERF-scrollback-snapshot** — `pane_content_from_damagegrid` (`jackin-capsule/src/tui/pane_snapshot.rs:182-194`) snapshots the whole 10k-row scrollback (owned `String` per cell) to read one cell / a few rows, and fires per mouse-motion event during Ctrl/Alt link-hover (`daemon/mouse_input.rs:653`). Add a borrowed row accessor + range-scoped snapshot. M, highest-leverage remaining perf item.
- **PERF-resize-clone** — `resize_grid` (`jackin-term/src/grid/write.rs:45-59`) allocates a fresh grid and deep-clones every retained cell on every `set_size` (both primary + alternate), with an O(rows×pool) arena scan; row storage is freed, not reused. Resize in place. M.
- **PERF-diag-double-parse** — `summary.rs:153-206` parses every JSONL line into an owned `serde_json::Value` and re-parses `detail` a second time; runs reach hundreds of MB. Typed borrowed deserialize + `RawValue` + reused line buffer. M.
- **PERF-benches-missing** — no bench covers resize, scrollback-snapshot/copy, `materialize_accounts`, or `summarize_reader`; the Phase-4 perf-budget lane can't gate them. Add one bench per hotspot. M.
- Low-leverage (recorded, likely not worth doing soon): usage-monitor whole-file re-read per poll (documented tradeoff), usage upsert without a cached prepared statement (verify turso caching first), `preserve_visible_rows_to_scrollback` double-clone.

### Tests

- **TEST-clock** — no injected clock; 97 raw `Instant/SystemTime/Utc::now()` sites make expiry/throttle/deadline logic untestable and force 10 sleep-based tests. Root blocker for all planned sim/failpoint/proptest work. Introduce a `Clock` seam (start with one consumer, e.g. `clipboard.rs` expiry). L, high leverage but touches many sites — scope to a first consumer.
- **TEST-launch-core** — `run_launch_core` has zero direct test; docker-run + wait-for-state are E2E-only. Drive it with `FakeDockerClient` (`runtime/test_support.rs:200`) for a happy path + a mid-pipeline-failure teardown assertion (pairs with plan 008). M.
- **TEST-displace-policy** — capsule multi-client displace / active-client policy has one reattach test; no “second client displaces, input routes to B” test. M. Natural home for the planned proptest-state-machine.
- **TEST-pty-recovery** — `session.rs:470-555` writer-lock-failure / closed-channel / PTY-EOF recovery paths are unexercised. Extend the mock PTY to return Err/EOF. M.
- **TEST-support-crate** — ≥3 `FakeRunner`, 2 `FakeDockerClient`, 4 `StubRunner`, duplicated `fake_with_outputs` across 8+ crates. Extract `jackin-test-support` with one canonical set. L. Roadmap Phase 3 item; sized here.
- **TEST-clipboard** (S) and **TEST-git-watch** (S-M) — untested clipboard image-chunk assembly / time-expiry and git/PR-watch throttle; both need TEST-clock first.

### Tech debt / architecture

- **DEBT-op-typed-error** — `op_cli::run_op_json` rewrites stderr into an `anyhow!` string that `oppicker::classify_probe_error_message` re-parses by substring across a crate boundary (~40 such `.contains()` classification sites workspace-wide). Define an `OpProbeError` thiserror enum. M. **Open decision:** which crate owns the enum (jackin-core vs a shared type) given the oppicker(pure)/jackin-env(side-effects) split — resolve before planning.
- **DEBT-anyhow-in-libs** — `anyhow` saturates library crates (runtime 39 files, console 35, capsule 22…); only 5 of 21 have any `thiserror`. Convert foundational crates first (`jackin-core` 8, `jackin-env` 12). L, phased.
- **DEBT-devdep-cycle** — `jackin-isolation` dev-deps `jackin-runtime` while runtime prod-deps isolation = a cycle through dev-deps; `jackin-term` dev-deps `jackin-diagnostics` (upward). Blocks a tier-graph arch gate. Move the shared test helpers into a neutral testkit crate. S-M. See also DX-arch-gate.
- **DEBT-type-complexity** — 47 `type_complexity` allows, 34 in `jackin-console` editor/settings view builders (19 in `editor/view/frame.rs` alone). Replace nested closure/tuple return types with named `FieldRow`/`SectionBuilder` structs. M.
- **DEBT-pub-surface** — foundational crates expose internals as broad `pub mod` (`jackin-core` 39 pub mods, `jackin-env` 11) rather than curated root re-exports, so every internal refactor risks a breaking change. Flip to `pub(crate)` + `pub use`. M per crate. Roadmap Phase 2.
- **DEBT-tui-half-layer** — the shared `jackin_tui::runtime` `View` trait has 3 thin implementors and 0 real dispatchers (each loop still calls `render` directly, self-documented as a follow-up). Either finish the loop-through-trait migration or drop the trait + context structs until a shared driver exists. M.
- **DEBT-newtypes** — `workspace`/`session`/`instance` ids cross crate boundaries as bare `&str` (58 workspace-name sites; adjacent swappable `&str` params in `launch_slot.rs`). Introduce `WorkspaceName`/`SessionId` newtypes (exemplars: `jackin-core::selector::RoleSelector`, `OpRef`). M. Roadmap Phase 2.

### DX / tooling (agent-contributor leverage)

- **DX-gate-json** — 0/8 xtask gates emit `--format json` or GitHub `::error file=,line=` annotations; an agent reading a red CI regexes prose. Add a shared human|json|github report helper + a clippy/rustc problem matcher. L. Roadmap “diagnostics are prompts”; baseline = zero.
- **DX-gate-messages** — bring the six weaker xtask gates up to the `lint tests` bar (each failure states the exact rerun command + fix); `lint tests` is the template. S.

## Direction findings (maintainer's call — options, not defects)

Forward-looking bets grounded in repo evidence, presented for the maintainer to weigh. Not ranked against the fixes above; each would become a design/spike plan, not a build-everything plan.

1. **Finish the Apple Container backend into a Docker-Desktop-free macOS tier.** A real `ContainerBackend` trait with `DockerBackend` + `AppleContainerBackend` impls and config surface already exist; `AppleContainerBackend::finalize` is a Phase-0 stub. `APPLE_CONTAINER_RESEARCH.md` says apple/container 1.0.0 shipped and the code targets v0.11.0, missing 1.0.0's `--publish-socket` and `--internal` networks. The only true blocker is Phase-0 validation on macOS 26 ARM hardware. Highest strategic leverage; the code is ~80% there. L.
2. **Agent-to-agent orchestration (“Conductor view”).** The capsule multiplexer already owns every agent's PTY, tracks per-session state, and records agent/provider/codename; a dedicated design page argues “mostly reuse.” Turns jackin❯ from “safe place to run one agent” into “visible control room for a team of agents.” L; substrate exists.
3. **Budget enforcement (spend caps), not just telemetry.** `jackin-usage` meters cost but explicitly cannot enforce a per-task/day cap (grep for `budget|enforce|kill` is clean). Trigger (status authority) + actuator (daemon/capsule) exist. With parallel autonomous agents, runaway spend is the top operational risk. M.
4. **First-party eval / outcome-scoring harness.** jackin❯ uniquely runs the same task across runtime × provider × role but measures only cost, never success. Cost telemetry + `jackin-dev pr` verification already exist to join. Category-defining for a multi-runtime tool and the signal a Conductor view needs. M-L.
5. **Local/self-hosted model as a first-class provider with hybrid routing.** Provider catalog + per-session model override + the DinD sidecar pattern a local engine would mirror all exist; fits the isolation/privacy thesis and powers cheap workers for #2. M.
