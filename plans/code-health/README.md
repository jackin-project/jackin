## Direction findings (maintainer's call — options, not defects)

Forward-looking bets grounded in repo evidence, presented for the maintainer to weigh. Not ranked against the fixes above; each would become a design/spike plan, not a build-everything plan.

1. **Finish the Apple Container backend into a Docker-Desktop-free macOS tier.** A real `ContainerBackend` trait with `DockerBackend` + `AppleContainerBackend` impls and config surface already exist; `AppleContainerBackend::finalize` is a Phase-0 stub. `APPLE_CONTAINER_RESEARCH.md` says apple/container 1.0.0 shipped and the code targets v0.11.0, missing 1.0.0's `--publish-socket` and `--internal` networks. The only true blocker is Phase-0 validation on macOS 26 ARM hardware. Highest strategic leverage; the code is ~80% there. L.
2. **Agent-to-agent orchestration (“Conductor view”).** The capsule multiplexer already owns every agent's PTY, tracks per-session state, and records agent/provider/codename; a dedicated design page argues “mostly reuse.” Turns jackin❯ from “safe place to run one agent” into “visible control room for a team of agents.” L; substrate exists.
3. **Budget enforcement (spend caps), not just telemetry.** `jackin-usage` meters cost but explicitly cannot enforce a per-task/day cap (grep for `budget|enforce|kill` is clean). Trigger (status authority) + actuator (daemon/capsule) exist. With parallel autonomous agents, runaway spend is the top operational risk. M.
4. **First-party eval / outcome-scoring harness.** jackin❯ uniquely runs the same task across runtime × provider × role but measures only cost, never success. Cost telemetry + `jackin-dev pr` verification already exist to join. Category-defining for a multi-runtime tool and the signal a Conductor view needs. M-L.
5. **Local/self-hosted model as a first-class provider with hybrid routing.** Provider catalog + per-session model override + the DinD sidecar pattern a local engine would mirror all exist; fits the isolation/privacy thesis and powers cheap workers for #2. M.
