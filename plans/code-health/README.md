## Findings considered and deferred (not yet planned ŌĆö recorded so they aren't re-audited)

High-value findings that are real but were not turned into first-wave plans

### Security
- **SEC-unify-redactors** ŌĆö three inconsistent secret-key predicates exist
  (`jackin-diagnostics::redact` regex, `secret_scrub::is_secret_key` substring,
  `shell_runner::is_sensitive_arg_key`); they disagree (e.g. none catches
  `GROK_DEPLOYMENT_KEY`), and the weakest guards the OTLP sink. Unify onto the
  `is_credential_env_key` predicate 005 introduces. M. Do after 002 + 005.
- **SEC-emit-redaction** ŌĆö `emit_debug_line`/`emit_compact_line`/`emit_operator_notice`
  (`jackin-diagnostics/src/logging.rs:177-231`) and `shell_runner do_capture`
  (`:503-511`) do not redact; safe only by unenforced `capture_secret` caller
  discipline. Pipe them through `redact_text` or a type marker. S.
- **SEC-socket-nonce** ŌĆö the capsule attach/control socket is authed only by uid
  + `0600` (`jackin-capsule/src/socket.rs:108-117`); a same-uid in-container
  process can inject into the operator's PTY / pop credential prompts. Add a
  per-launch connection nonce the container uid can't read. M, hostŌåöcapsule wire
  change. Partly an inherent container-local-IPC tradeoff already acknowledged
  in-code.

### Correctness / bugs
- **BUG-color-hex-slice** ŌĆö `host_colors.rs:161-166` (duplicated in
  `jackin-tui` and `jackin-runtime`) slices a `#RRGGBB` OSC reply by byte under a
  byte-length guard; a 6-byte multibyte string panics. Guard with `is_ascii()` or
  parse from `as_bytes()`; dedupe the two copies. S. Low reachability (operator's
  own terminal reply) but a concrete `string_slice` instance.
- **BUG-pty-unbounded-channel** ŌĆö capsule `event_tx/event_rx`
  (`daemon.rs:202-203`) is unbounded; a runaway agent (`yes`) has no PTY
  backpressure and can inflate capsule RSS. Investigate depth under a flood soak
  before switching to a bounded/coalescing channel. LOW-MED, investigate-first.

### Performance
- **PERF-scrollback-snapshot** ŌĆö `pane_content_from_damagegrid`
  (`jackin-capsule/src/tui/pane_snapshot.rs:182-194`) snapshots the whole 10k-row
  scrollback (owned `String` per cell) to read one cell / a few rows, and fires
  per mouse-motion event during Ctrl/Alt link-hover
  (`daemon/mouse_input.rs:653`). Add a borrowed row accessor + range-scoped
  snapshot. M, highest-leverage remaining perf item.
- **PERF-resize-clone** ŌĆö `resize_grid` (`jackin-term/src/grid/write.rs:45-59`)
  allocates a fresh grid and deep-clones every retained cell on every `set_size`
  (both primary + alternate), with an O(rows├Śpool) arena scan; row storage is
  freed, not reused. Resize in place. M.
- **PERF-diag-double-parse** ŌĆö `summary.rs:153-206` parses every JSONL line into
  an owned `serde_json::Value` and re-parses `detail` a second time; runs reach
  hundreds of MB. Typed borrowed deserialize + `RawValue` + reused line buffer. M.
- **PERF-benches-missing** ŌĆö no bench covers resize, scrollback-snapshot/copy,
  `materialize_accounts`, or `summarize_reader`; the Phase-4 perf-budget lane
  can't gate them. Add one bench per hotspot. M.
- Low-leverage (recorded, likely not worth doing soon): token-monitor whole-file
  re-read per poll (documented tradeoff), usage upsert without a cached prepared
  statement (verify turso caching first), `preserve_visible_rows_to_scrollback`
  double-clone.

### Tech debt / architecture
- **DEBT-command-seam** ŌĆö 19 files call `Command::new("gh"|"op"|"docker"|"git")`
  directly, bypassing the `CommandRunner` port's `capture_secret` redaction; the
  port has no timeout/retry (only `op_cli::run_op_with_timeout` does). Extend
  `RunOptions` with timeout/retry/secret; migrate sites family-by-family. L. High
  value (security + hang-resistance); `SharedCommandRunner` (currently
  `#[expect(dead_code)]`) is its not-yet-wired half.
- **DEBT-op-typed-error** ŌĆö `op_cli::run_op_json` rewrites stderr into an
  `anyhow!` string that `oppicker::classify_probe_error_message` re-parses by
  substring across a crate boundary (~40 such `.contains()` classification sites
  workspace-wide). Define an `OpProbeError` thiserror enum. M. **Open decision:**
  which crate owns the enum (jackin-core vs a shared type) given the
  oppicker(pure)/jackin-env(side-effects) split ŌĆö resolve before planning.
- **DEBT-anyhow-in-libs** ŌĆö `anyhow` saturates library crates (runtime 39 files,
  console 35, capsule 22ŌĆ”); only 5 of 21 have any `thiserror`. Convert
  foundational crates first (`jackin-core` 8, `jackin-env` 12). L, phased.
- **DEBT-auth-path-contract** ŌĆö agent auth/config-dir path table duplicated
  across `jackin-capsule/runtime_setup.rs:25-105` (container side) and
  `jackin-instance/auth.rs` + `lib.rs` (host side); drift = silent
  auth-fallback-to-login. Extract one `AgentAuthLayout` per agent into
  `jackin-core`. M.
- **DEBT-github-url** ŌĆö GitHub URL normalization/credential-stripping siloed in
  the `jackin-console` leaf (`mount_info.rs`, scp-parse written twice) plus
  hand-rolled copies in config/capsule. Lift `GithubRemote::parse` into a low
  crate. S-M.
- **DEBT-devdep-cycle** ŌĆö `jackin-isolation` dev-deps `jackin-runtime` while
  runtime prod-deps isolation = a cycle through dev-deps; `jackin-term`
  dev-deps `jackin-diagnostics` (upward). Blocks a tier-graph arch gate. Move the
  shared test helpers into a neutral testkit crate. S-M. (See also DX-arch-gate.)
- **DEBT-type-complexity** ŌĆö 47 `type_complexity` allows, 34 in
  `jackin-console` editor/settings view builders (19 in `editor/view/frame.rs`
  alone). Replace nested closure/tuple return types with named `FieldRow`/`SectionBuilder`
  structs. M.
- **DEBT-pub-surface** ŌĆö foundational crates expose internals as broad `pub mod`
  (`jackin-core` 39 pub mods, `jackin-env` 11) rather than curated root
  re-exports, so every internal refactor risks a breaking change. Flip to
  `pub(crate)` + `pub use`. M per crate. (Roadmap Phase-2.)
- **DEBT-tui-half-layer** ŌĆö the shared `jackin_tui::runtime` `View` trait has 3
  thin implementors and 0 real dispatchers (each loop still calls `render`
  directly, self-documented as a follow-up). Either finish the loop-through-trait
  migration or drop the trait + context structs until a shared driver exists. M.
- **DEBT-newtypes** ŌĆö `workspace`/`session`/`instance` ids cross crate boundaries
  as bare `&str` (58 workspace-name sites; adjacent swappable `&str` params in
  `launch_slot.rs`). Introduce `WorkspaceName`/`SessionId` newtypes (exemplars:
  `jackin-core::selector::RoleSelector`, `OpRef`). M. (Roadmap Phase-2.)

## Direction findings (maintainer's call ŌĆö options, not defects)

Forward-looking bets grounded in repo evidence, presented for the maintainer to
weigh. Not ranked against the fixes above; each would become a design/spike plan,
not a build-everything plan.

1. **Finish the Apple Container backend into a Docker-Desktop-free macOS tier.**
   A real `ContainerBackend` trait with `DockerBackend` + `AppleContainerBackend`
   impls and config surface already exist; `AppleContainerBackend::finalize` is a
   Phase-0 stub. `APPLE_CONTAINER_RESEARCH.md` (52K, adversarially verified) says
   apple/container 1.0.0 shipped and the code targets v0.11.0, missing 1.0.0's
   `--publish-socket` (tailor-made for jackin-exec) and `--internal` networks. The
   only true blocker is Phase-0 validation on macOS 26 ARM hardware. Highest
   strategic leverage; the code is ~80% there. L (front-loaded on empirical
   validation).
2. **Agent-to-agent orchestration ("Conductor view").** The capsule multiplexer
   already owns every agent's PTY, tracks per-session state, and records
   agent/provider/codename; a dedicated design page argues "mostly reuse." Turns
   jackinŌØ» from "safe place to run one agent" into "visible control room for a
   team of agents" ŌĆö the moat the Apple research itself names. L; substrate
   exists.
3. **Budget enforcement (spend caps), not just telemetry.** `jackin-usage` meters
   cost but explicitly cannot enforce a per-task/day cap (grep for
   `budget|enforce|kill` is clean). Trigger (status authority) + actuator
   (daemon/capsule) exist. With parallel autonomous agents, runaway spend is the
   top operational risk. M.
4. **First-party eval / outcome-scoring harness.** jackinŌØ» uniquely runs the same
   task across runtime ├Ś provider ├Ś role but measures only cost, never success.
   Cost telemetry + `jackin-dev pr` verification already exist to join. Category-
   defining for a multi-runtime tool and the signal a Conductor view needs. M-L.
5. **Local/self-hosted model as a first-class provider with hybrid routing.**
   Provider catalog + per-session model override + the DinD sidecar pattern a
   local engine would mirror all exist; fits the isolation/privacy thesis and
   powers cheap workers for #2. M.
6. **Protect the agent/repo/credentials from untrusted content (MCP governance +
   prompt-injection boundary).** jackinŌØ»'s isolation protects the host from the
   agent, not the agent (and the mounted repo + forwarded creds) from malicious
   content; egress is allow-listed, so a prompt-injected agent can still exfil to
   an allowed destination, and nothing inspects the MCP protocol. Start with an
   MCP-call audit trail + human approval for write/execute tools. L, more
   speculative.
