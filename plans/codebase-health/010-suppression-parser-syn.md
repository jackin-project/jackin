## Why this matters

Roadmap Rust-enforcement item 6: "Make suppression measurement trustworthy before using it as a ratchet. Parse Rust attributes with a syntax-aware parser…; add comment, string, raw-string, multiline, and nested-syntax negative cases; generate the baseline and ratchet from real attributes." The current hand-rolled byte scanner splits attribute bodies on commas BEFORE isolating the `reason` string, so prose words from comma-containing reasons leak into the enforced ratchet as fake lint names — `ratchet.toml` today gates on keys like `and@jackin-xtask=2`, `cargo@jackin-xtask=2`, `before@jackin-usage=1`, `not@jackin=2`. The gate is enforced at PR time (`cargo xtask lint --strict` runs `suppressions::enforce()` + `ratchet::enforce()`), so CI is defending meaningless numbers, and any reason-string edit that adds/removes a comma silently loosens or breaks it. The scanner also counts attribute-shaped text inside string literals (a test fixture string is grandfathered as a "real" bare allow in `ratchet.toml`), and `code-health-baseline.toml`'s `[suppressions.by_lint]` is polluted with English words (`"the" = 8`, `"not" = 19`, `"and" = 14`).

```rust
fn parse_lint_list(body: &str) -> Vec<String> {
    let mut lints = Vec::new();
    for part in body.split(',') {
        let part = part.trim();
        if part.is_empty() || part.starts_with("reason") { continue; }
        let token = part.split_whitespace().next().unwrap_or("").trim_matches(|c: char| c == '(' || c == ')');
        if token.is_empty() || token.contains('=') { continue; }
        if token.chars().all(|c| c.is_ascii_alphanumeric() || c == '_' || c == ':') { lints.push(String::from(token)); }
    }
    lints
}
```

## Steps

### Step 1: Rewrite the scanner on syn

Replace `parse_suppression_attrs`/`find_matching_paren`/`parse_lint_list` with a `syn`-based pass: `syn::parse_file` each `.rs`, walk items/attributes (including inner attributes and attributes on items nested in modules/impls/fns — use `syn::visit::Visit`), collect `allow`/`expect` `Meta::List` attributes, extract lint paths structurally (each `Meta::Path` element), and read `reason` as a `Meta::NameValue`. `cfg_attr`-wrapped allow/expect: unwrap one level and record. Files that fail to parse: hard error naming the file (never silently skip).

**Verify**: `cargo nextest run -p jackin-xtask -E 'test(/health::tests/)'` → existing positive tests pass unchanged.

### Step 2: Negative test table

Add table-driven negatives: attribute text inside `//` comment, `/* */` block, normal string, raw string `r#"…"#`, char literal, doc comment; comma-inside-reason yields exactly the real lint(s); multiline attributes; `#[cfg_attr(test, allow(dead_code))]`. Each asserts exact expected (lint, reason) pairs.

**Verify**: `cargo nextest run -p jackin-xtask -E 'test(/health::tests/)'` → all pass.

### Step 3: Regenerate baseline + ratchet

Run the print/regenerate commands; update `ratchet.toml` suppression families (fake keys — `and@…`, `cargo@…`, `gh@…`, `not@…`, `before@…`, `kept@…`, `measured_n@…`, `redirectable@…`, `synchronous@…`, `git@…` — disappear; real per-lint counts refresh); delete the `jackin-xtask bound = 1` bare-allow grandfather row (the fixture string no longer counts); run `cargo xtask health --write-baseline` and verify `[suppressions.by_lint]` now contains only tokens matching `^(clippy::)?[a-z0-9_]+$` that are plausible lint names (spot-check against `Cargo.toml` lint tables).

**Verify**: `cargo xtask lint suppressions` → exit 0; `cargo xtask lint ratchet` → exit 0; `grep -c '"and"\|"not"\|"the"' code-health-baseline.toml` → 0.

### Step 4: Reason-policy proof

Add a test proving the gate enforces the reason policy end to end: a fixture source (in-test string parsed via the new syn path, not a checked-in file) with a bare `#[allow(dead_code)]` is reported as bare; with `#[expect(dead_code, reason = "x")]` is not. This is the roadmap's "prove that `cargo xtask lint suppressions` reports the enforced reason policy".

**Verify**: `cargo nextest run -p jackin-xtask` → pass; `cargo xtask ci --fast` → exit 0.
