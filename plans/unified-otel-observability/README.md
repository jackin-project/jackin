## Cross-plan invariants (bind every executor)

- The extension registry is CLOSED. No new `jackin.*`/`parallax.*`/ad-hoc attribute, event, span, or metric name — additions go through the schema registry (plan 001) and, for new semantic domains, through a roadmap-contract change first.
- No lifetime spans: process, invocation, session, screen visit, daemon, focus, stream — ever.
- Privacy allowlist-first: paths, names, labels, argv, URLs with queries, credentials, PTY/terminal content, prompts, and user text are prohibited everywhere; when in doubt, the field stays out.
- ONE branch (`feature/unified-otel-observability`), ONE pull request for the entire item — no per-plan branches, no PR splitting.
- Repo hard rules apply to every commit: never commit to `main`; sign `-s`; push after every commit; Conventional Commits; pre-release breaking changes without shims; brand `jackin❯` in prose.
- Verification gates live in each plan; the merge-readiness gate is `cargo xtask ci` (or `--fast` locally).

## Findings considered and rejected

- **Keep `parallax.run.id` as an ecosystem-compat key** — rejected by the roadmap contract itself ("The legacy `parallax.run.id` and every `jackin.*` telemetry key are removed"); plan 013 carries an explicit operator-confirmation STOP because a prior project memory recorded the opposite intent.
- **Route capsule telemetry through the host (relay/proxy)** — rejected: the contract mandates independent direct export per process; `network.mode=none` disables capsule egress instead.
- **Local file fallback when Parallax is unreachable** — rejected: bounded in-memory queues drop on saturation by design; no spool, no crash-durability channel (roadmap "Out of scope" and OTLP-spec basis).
- **Using an OpenTelemetry Collector in tests or as a dev dependency** — rejected: no Collector configuration belongs in the repository; plan 014 builds a minimal in-repo OTLP test receiver instead.
