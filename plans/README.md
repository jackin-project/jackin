below unless dependencies say otherwise. Each executor: read the plan fully

## Findings considered and rejected / deferred (so nobody re-audits them)

- **op:// value caching on disk** — rejected: resolved 1Password secrets must
  not persist outside `op`; the correct fix is overlap (Plan 003) —
  the 120 s per-read timeout (`op_cli.rs:12`, widened by #709) remains the
  worst-case bound.
- **Launch-on-previous-SHA image while new role commit builds in background**
  (would make role-commit launches instant) — deferred as a product decision:
  it runs a session on an older role definition; needs an opt-in flag design.
  The `RefreshInBackground` machinery is the natural extension point
  (`ImageDecision`, `image_decision.rs`).
- **Provision only the selected agent (not all supported)** — deferred design
  tradeoff: sibling tabs exec sibling CLIs in the same container
  (`launch_core.rs:222-228` documents why). Needs lazy per-tab provisioning.
- **Per-agent Dockerfile stage isolation** (one agent bump rebuilds sibling
  layers + default-home tail) — deferred; multi-stage `COPY --link=from`
  redesign (audit PERF-07). Plan 002 removes the worst case (capsule bump).
- **Role-base build-context full copy → hardlink/tar stream** (audit PERF-06,
  `derived_image.rs:740`, `copy_dir_all`) — valid S/M win on role-base builds;
  not planned this round, fold into a build-path follow-up.
- **decide-path `list_image_tags`+`inspect_image_labels` merge** (2→1 daemon
  round-trip) — valid, small; do opportunistically with 001.
- **Repo-lock narrowing across the network fetch** — deferred; Plan 004's TTL
  removes the common contention window first.
- **Gate role `docker run` on sidecar-start instead of TLS-ready** — deferred:
  agent could race a not-ready dockerd; needs an in-container lazy gate.
- **Capsule daemon boot micro-costs** (git/gh probes before socket listener,
  `jackin-capsule/src/daemon.rs:858` + `git_context.rs:46-74`; post-run
  firewall/sudo execs serial before attach, `launch_runtime.rs:960-995`) —
  real but ~100–500 ms class; measured capsule stage is already ~0.5 s. Listed
  in Plan 008 maintenance notes as follow-ups.
- **Console instance-refresh `docker ps` subprocess + thread fan-out**
  (`console/services.rs:391,489`) — off the launch action; separate cleanup.
- **`jackin-manifest` double TOML parse, repeated `canonicalize`, version-const
  reparse** (audit repo PERF-04) — trivial S cleanup, bundle with any manifest
  PR.
- **Bugs found while auditing, out of launch-speed scope — file separately:**
  - Claude auth provisioner missing the no-churn guard → concurrent
    foreground-prepare / sibling-prewarm inode swap can race `docker create`
    and un-auth a sibling tab (`jackin-instance/src/auth.rs:514-518` vs the
    guarded single-file path at `:989-1006`); also
    `read_to_string(...).unwrap_or_else("{}")`-style fallback at `:1077` can
    clobber `account.json` on transient read error.
  - Worktree leak when sidecar fails mid-`join!` — author-acknowledged
    TODO(launch-worktree-leak-on-sidecar-fail), `launch_core.rs:807-810`.
  - Transient `git rev-parse HEAD` failure silently degrades to the mutable
    un-suffixed image tag + full rebuild (`runtime/image.rs:1707-1738`,
    `naming.rs` bare-tag fallback).
  - `wait_for_dind` classifies cert errors by matching an exec error string
    (`attach.rs:1113`) — fragile coupling.
  - Stale doc on dead `runtime_mount_binary_path` (removed by Plan 005 Step 6).
