# Implementation Plans — Telemetry Ecosystem Program

Generated by the improve skill on 2026-07-03 (deep audit of the logging/tracing/metrics ecosystem; planned at commit `5d3661cff`). Execute in the order below unless dependencies say otherwise. Each executor: read the plan fully before starting, honor its STOP conditions, and update your row when done.

Audit inputs: nine parallel read-only audits over `jackin-diagnostics`, host call sites, capsule call sites, error semantics, metrics/usage, config surface, test coverage, docs/direction, plus the live-backend findings in `docs/content/docs/reference/research/agent-telemetry/parallax-observability-findings.mdx` (treated as hypotheses; every claim used here was re-verified against code). Parallax-backend counterpart plans live in the parallax repo: `plans/019-021` there (fingerprint normalization, spool retention, anchored-query fix).

Design direction the program encodes (validated against tracing 0.1 / tracing-opentelemetry 0.33 / opentelemetry-rust 0.32 / OTel spec ≥1.53): one instrumentation model — `tracing` spans and structured events are the single emission API; console/file prefixes are render-boundary formatting; per-layer `EnvFilter`s (allowlist) scope each sink; severity carries truth (content, not tier); payload bytes never cross the export boundary; high-frequency facts become metrics, not log rows; `otel.name`/span links give readable traces; the OTLP backend and the local JSONL file are sinks of the same events, never parallel systems.

## Findings considered and rejected / deferred (so nobody re-audits them)

- **Two-tier `clog!`/`cdebug!` console+file design** — not a finding; documented ENGINEERING.md hard rule. The program changes export semantics, not the tier concept.
- **`run.rs` presentation helpers (`section`/`PendingRow`, run.rs:1024-1114)** — layering smell, but a documented deliberate inline (avoids L1→L3 dep); revisit only with a broader crate-layering pass. Not planned.
- **clog!/cdebug! macros living in `jackin-usage`** (ARCH-02) — real layering inversion, but relocating 293 call sites' macro origin is churn with no behavior gain while plans 003/005/006 are rewriting the same macros' bodies. Re-evaluate after 006 lands; a move then is mechanical.
- **`launch_stage` custom-kind stages bypassing duration/foreground-wait machinery** — folded into plan 007's guard redesign rather than a separate fix.
- **OTLP batch-queue sizing / sampler** — deferred until after 002 removes the overflow source (noted in 011 maintenance).
- **Usage-refresh flock held across DB write + JSON fsync** (audit PERF-05) — real, but cross-container coordination semantics need a dedicated design; deliberately not in 012.
- **`telemetry_store.rs` `#[cfg(test)]` read-path half** — scaffolding for a roadmap host-daemon store; decide there, not in this program.
- **Substring-based `launch_failure_title` classifier** — superseded in telemetry by `error_type` (plan 003); the display path keeps it until a UI pass.
- **`capture` vs `capture_secret` type enforcement (newtype)** — flagged as follow-up in plan 005 maintenance.
- **Host resource lacking `session.id`** — host has no session concept today; revisit if host-side session grouping becomes a need.
- **`operator_env` no-values invariant test** — covered by plan 005's redaction tests at the export boundary (characterization for that specific renderer welcome but not planned).
