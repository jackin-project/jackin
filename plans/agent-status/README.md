# Agent Runtime Status — root-cause plans

Why the capsule tab status icon "never changes" — and the fixes, judged by correctness and structural
root cause (not effort). Generated by a deep read-only audit (5 parallel investigators: render wiring,
compute path, detection packs, herdr reference, web research) on 2026-07-03, against commit `5d3661cff`
on `main`. Reference implementation studied: **herdr** (`github.com/ogulcancelik/herdr`, AGPL-3.0 — local
copy at `/Users/donbeave/Projects/github/herdr`; **approach only, never code**, jackin is Apache-2.0).

Operator report: the per-tab status glyph in `jackin-capsule` is effectively frozen — it does not reflect
blocked / working / idle transitions. Operator goal: **all four states visible on the tab icon with zero
config, for every supported agent** — 🔴 blocked, 🟡 working, 🔵 done, 🟢 idle (claude, codex, amp, kimi,
opencode, **and grok**).

## The bug is not one bug — it is four independent layers, each of which alone freezes the tab

The status pipeline is `identify agent → collect evidence → arbitrate → author session.state → render
tab glyph`. **Every** stage has a structural defect that can, on its own, make the tab appear static.
Fixing only one leaves the symptom. Per the operator's root-cause directive ("every bug is evidence the
architecture *permits* it; prefer removing the structural condition over a symptom patch"), each plan
targets the enabling condition, not the instance.

| Layer | Structural condition that permits the bug | Consequence | Plan |
|---|---|---|---|
| **Render** | Three hand-maintained state vocabularies (`AgentState`(5) → `VisibleAgentState`(4) → `TabGlyph`(3)) connected by lossy `match`es with no totality enforcement. `TabGlyph` has **no slot** for working/idle — they paint blank by construction. `Unknown → Idle` fold. | Even perfect detection can never show working/idle; the tab only ever changes on blocked/done (the two rarest moments). This alone reproduces the symptom. | **001** |
| **Identification** | `identify_agent` unwraps the node/bun shim **only for Claude**; every other wrapped agent → `None`. No generic runtime/shell unwrap; no exhaustiveness over `Agent::ALL`. | Any npm-distributed agent (opencode/codex/amp/kimi/grok) launched via a node shim identifies as nothing → no screen match *and* invalidated hook authority → permanently dark. | **002** |
| **Compute cadence** | The sole state-authoring call (`advance_status`) rides the **last seat of a `biased` tokio select**, ranked below the unbounded PTY-output arm. The "1 Hz floor" is a prose invariant with no structural backing. | A streaming/animated agent keeps the output arm `Ready` → the state ticker is never polled → status freezes **exactly while the agent is busy**. | **003** |
| **Compute redundancy** | For claude/codex the design advertises 4 evidence sources but 3 are inert: hooks are identity-only (Decision 0a → never author state), **OSC-133 has no emitter at all** (dead source, and a sticky no-TTL level that would pin state if wired), physics can only express working/unknown. All blocked/idle/done funnel through one fragile screen pack. | Any drift in the screen pack silently disables all non-working status for the primary agents. | **004** |
| **Detection reality** | Packs are "validated" against fixtures **authored from the same guessed strings** (circular); the one drift guard (`accepts_cli_version`) is **dead code** (only a test caller). A pack can be 100% wrong with a green suite. | kimi/amp/opencode/claude-idle pack literals are fabricated placeholders that don't match the real TUIs → those agents never fire blocked/idle. | **005**, **007** |
| **Detection coverage** | `Agent::ALL` and the pack/reporter asset sets are unenforced **parallel lists**; a failed embedded-pack load zeroes the whole registry silently; unmatched agent → `Unknown` with no operator-visible signal. | grok shipped with zero detection; any future agent added to the enum is silently undetectable. | **006** |
| **Testability** | `advance_status` (the assembly of collect→arbitrate→publish) has **zero** test coverage; unit tests inject a synthetic `EvidenceSnapshot` past the broken seam. Physics/authority is `/proc`-gated, so on the dev platform (macOS) it is never exercised. | Wiring breaks in every layer above ship green. | **008** |

## Detection is not its own crate today (verified) — plan 011 isolates it

Checked at `5d3661cff`: the workspace `Cargo.toml` has **no** `jackin-agent-status` member; all detection
lives inside the 50K-line `jackin-capsule` crate at `crates/jackin-capsule/src/agent_status/`. The good news
— the detection core is **already sync and pure** (no tokio/async) and imports **nothing** from capsule
internals (no `jackin-term`, no `Session`/`tui`/`daemon`); it depends only on `jackin-protocol` (wire types),
`jackin-core` (`Agent`), and `regex`/`semver`/`toml`. `Session::advance_status` already assembles a plain
`EvidenceSnapshot` and calls the pure `arbitrate`/`apply_watchdog`. So extracting an independent
`jackin-agent-status` crate is a low-risk mechanical move (plan 011) that gives detection its own isolated,
independently-testable, reusable home — and becomes the owner for the render vocabulary source (001), the
packs (005/007), the remote-pack channel (010), and the reporter gating (009).

first — but note it will surface the detection gaps (a working agent showing 🟡 requires working detection
to actually fire). **Structurally, extract the detection crate (011) early** — the detection core is already
sync/pure, and moving 003/004/005/007 into a clean crate beats moving them twice. The dependency spine:

## Licensing — herdr is AGPL; we borrow the concept, never the code (binding rule for every plan)

**herdr** is dual-licensed **AGPL-3.0-or-later** or commercial (`herdr/LICENSE`); **jackin** is **Apache-2.0**
(`LICENSE`, `Cargo.toml`). These are incompatible for code reuse — AGPL copyleft would attach to any derivative
work. This plan set relies on the **idea/expression** boundary: copyright protects the *expression* (source
code, its structure, and authored creative artifacts), **not** ideas, algorithms, methods, or functional
approaches. Studying how herdr detects status and independently implementing that *approach* is allowed and is
**not** a derivative work, so AGPL's terms do not reach jackin. Rules every executor must follow:

- **Do NOT copy herdr source code** — not verbatim, not line-by-line translated, not lightly edited. Read it to
  understand the *behavior*, then write jackin's own implementation from scratch.
- **Do NOT copy herdr's authored artifacts** — its manifest/rule TOML files and its committed fixture files are
  creative expression under AGPL; author jackin's own packs (plans 006/007) and capture jackin's own fixtures
  (plan 005). Never commit a herdr file into this tree.
- **Facts and functional constants are fine** — e.g. the 700 ms / 3-confirmation debounce values jackin already
  uses (`policy.rs:10-15`, honestly attributed "Herdr-validated bound"): a numeric parameter is a fact/idea, not
  protectable expression.
- **Keep the clean-room posture** the project already states — the design doc records that jackin's implementation
  was "written from scratch" from a concept review, "not source reuse." Preserve that: behavior → plan in our own
  words → implement from the plan.

Verified at `5d3661cff`: jackin's code contains **no** herdr source, no herdr references, and no copied
artifacts (grep of `crates/` + `docker/` is clean); the debounce-constant attribution comment is the only
mention, and that is an honest fact-attribution, not copied code. This is a factual/technical review, **not
legal advice** — for anything ambiguous about a specific artifact, capture/author jackin's own rather than reuse.

## How herdr does it (the reference), and where jackin differs

herdr = **zero-config observer**: foreground-process-group identification with generic wrapper unwrap
(node/bun/python/sh/bash/zsh/fish/cmd/powershell, with eval/module-flag guards); reads the **last ~24 rows
of the current rendered grid** (not scrollback); per-agent TOML rules with priority; the **agent's own OSC
title spinner** is the dominant working/idle signal for claude/codex, backed by screen-chrome for blocked;
`visible_blocker` overrides weaker signals; working→idle held through 3 confirmations / 700 ms cap; `done` =
raw idle on an unseen pane; **18 always-bundled manifests** (grok included) that are *always* present as a
fallback regardless of CLI version.

**Where jackin is already even or ahead — do NOT "fix" these:** the `arbitrate` precedence is as principled
as herdr and already borrowed herdr's exact 700 ms / 3-confirmation debounce constants
(`policy.rs:10-15`, "Herdr-validated bound"); both read a *rendered grid* not raw PTY bytes (redraws
resolved); both prioritize waiting-over-busy; jackin's subagent/descendant roll-up is arguably ahead
(`subagents_active` + `descendant_process_count`). The gaps are upstream (identification, packs, render
vocabulary, cadence), not in arbitration quality.

**Where jackin can be strictly better than herdr:** it owns the image/container, so it can install a
first-party semantic reporter (herdr can only *offer* optional integrations). Today that advantage is used
for only 2 of 6 agents (opencode plugin, amp mapping). Plan 009 leans into it.

## Considered and rejected

- **Active CSI 6n cursor-position probing (RESEARCH-08)** — rejected. Writing CPR queries into a full-screen
  agent TUI's PTY risks corrupting its input stream and racing its redraws; the readiness signal is weaker
  than the OSC/hook/screen edges both projects already have. Passive observation is the correct model; spend
  the effort on 002/004/009. Recorded so it isn't re-proposed.
- **Removing `biased` alone (as the 003 fix)** — insufficient: a fair select still cannot guarantee a cadence
  floor under output saturation. 003 decouples state advancement from the I/O select instead.

## Confidence & what was not verified live

All findings were read from source and cross-checked against the herdr reference; the auditors could **not**
run the real 2026 agent CLIs to capture live screens, so DETECT-02/007's "these exact strings don't match"
rests on the herdr manifests + code reading, not a live capture. Plan 005 (real captured goldens) is the
step that closes that gap and must precede 007. `accepts_cli_version` dead-code, the biased-select ordering,
the missing OSC-133 emitter, the Claude-only node unwrap, grok's absent pack, and the 3-vocabulary render
drift were all directly verified.
