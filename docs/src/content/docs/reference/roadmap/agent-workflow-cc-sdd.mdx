## Problem

The current spec workflow uses `obra/superpowers` — a Claude Code plugin that stores specs in `docs/superpowers/specs/` outside the Starlight site. Specs are not browsable, not linked to code, and not version-controlled in a format accessible to all agents.

## Goal

Replace with `cc-sdd` (gotalab/cc-sdd) — a minimal spec-driven development harness using `.claude/commands/` files that are version-controlled in the repo and visible to all agents. Specs land in `docs/src/content/docs/internal/specs/` and become browsable at `jackin.tailrocks.com/internal/specs/`.

## What superpowers provides today

- Brainstorming skill → spec authoring
- Writing-plans skill → implementation plan
- TDD, debugging, review skills
- Specs at `docs/superpowers/specs/`

## cc-sdd equivalent

- `/spec` command → creates a spec MDX page at `docs/src/content/docs/internal/specs/`
- `/plan` command → creates an implementation plan
- `/execute` command → implements against the spec
- No external tooling dependency — just `.claude/commands/*.md` files in the repo

## Steps

1. Install cc-sdd: follow gotalab/cc-sdd setup (creates `.claude/commands/spec.md`, `plan.md`, `execute.md`).
2. Add a 5-line section to <RepoFile path="AGENTS.md" /> pointing to cc-sdd and the spec location.
3. Migrate existing `docs/superpowers/specs/` → `docs/src/content/docs/internal/specs/` (review each file; convert shipped features to spec MDX pages).
4. Remove superpowers plugin from Claude Code configuration.
5. Verify `/spec` works in a new Claude Code session.
