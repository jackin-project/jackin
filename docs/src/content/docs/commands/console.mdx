## Synopsis

```bash
jackin console [--debug]
```

Open the operator console ‚Äî an interactive terminal UI for picking a
workspace and agent, managing saved workspaces, and launching agents
into isolated containers.

Running `jackin` with no subcommand opens the same console when stdout
is attached to an interactive terminal that is at least 40x15 in size.
On non-interactive shells (piped output, CI, very small terminals) bare
`jackin` silently prints top-level help instead. `jackin console` makes
the intent explicit and errors out when the terminal cannot host the
TUI.

The important distinction is:

- **workspace** = which files are mounted
- **agent** = which tool profile is loaded

## How it works

`jackin console` opens directly to the workspace manager list. Select a
workspace with `‚Üë`/`‚Üì` and press `Enter` to launch. The manager list
**is** the opening view ‚Äî there is no separate workspace-picker stage.

If agent disambiguation is needed, a compact picker appears as an
overlay on the manager list. There is no separate full-screen
agent-selection screen.

## Workspace selection

The manager list shows two kinds of rows:

- **Current directory** ‚Äî a synthetic row that mounts your `cwd` at the same path inside the container
- **Saved workspaces** ‚Äî named definitions from `~/.config/jackin/config.toml`

If your current directory falls under a saved workspace's host `workdir` or one of its mounted host paths, that workspace is preselected.

## Agent disambiguation

Pressing `Enter` on a workspace row routes by eligible-agent count:

- **`default_agent` set** on the workspace ‚Üí launch immediately with that
  agent. No picker.
- **Exactly one eligible agent** (after the workspace's `allowed_agents`
  filter) ‚Üí launch immediately with it. No picker.
- **Multiple eligible agents and no default** ‚Üí an inline `Select Agent`
  picker overlays the manager list. Type to filter, `‚Üë`/`‚Üì` to navigate,
  `Enter` to commit, `Esc` to cancel.

The picker is anchored on the manager list ‚Äî cancelling drops you back
on the same workspace row with the launch unchanged.

## When to use the console vs `load`

`jackin console` is the simplified, intuitive TUI for everyday
workspace launches. [`jackin load`](/commands/load/) is the full-surface
CLI ‚Äî together with its neighbours (`hardline`, `eject`, `exile`,
`purge`, `workspace`, `config`) it exposes every flag and capability
jackin offers.

The console intentionally stays narrower. **New niche flags and
detail-level options land on the CLI first, and sometimes exclusively.**
The console surfaces only what fits into an intuitive flow ‚Äî treat it
as a fast path for the common cases, not as a visual alias for the CLI.

| | `jackin console` | `jackin load` |
|---|---|---|
| **Interface** | Visual TUI | Command-line arguments |
| **Best for** | Exploring options, first-time setup, day-to-day launches | Scripting, repeatable workflows, feature-specific flags |
| **Workspace selection** | Interactive picker | Explicit argument |
| **Agent selection** | Interactive picker | Explicit argument |
| **Feature surface** | Simplified ‚Äî common flows only | Complete ‚Äî every flag jackin exposes |
| **Where new niche flags land** | Rarely | First, and sometimes exclusively |

<Aside type="tip">
The older spelling `jackin launch` still works but is deprecated and
emits a warning. It delegates to the same console handler as
`jackin console`.
</Aside>

## Workspace editor ‚Äî Secrets tab

The Secrets tab in the `jackin console` workspace editor manages env vars at
two scopes:

- **Workspace env** ‚Äî applies to every agent launch in this workspace,
  stored under `[workspaces.<name>.env]`.
- **Per-agent overrides** ‚Äî applies only when a specific agent is launched in
  this workspace, stored under `[workspaces.<name>.agents.<agent>.env]`.

Values are stored as-typed. Use `op://...` for 1Password references, `$VAR`
or `${VAR}` for shell-variable substitution, or a literal string.
jackin' resolves them at launch via `operator_env`.

| Key | Action |
|---|---|
| `Enter` on key row | Edit the value in a modal text input |
| `D` on key row | Delete the key (single confirmation) |
| `A` | Add a new key/value pair to the focused section |
| `M` | Toggle value masking (masked by default; resets on tab exit) |
| `P` on key row | Open 1Password picker; commits `op://...` directly to the key |
| `P` on `+ Add` row | Open 1Password picker; on commit, prompt for key name |
| `‚Üí` or `Enter` on section header | Expand an agent override section |
| `‚Üê` on section header | Collapse an agent override section |

### 1Password picker

Pressing `P` on a Secrets-tab row opens a three-pane picker that browses
your 1Password vaults and items. Navigate with `‚Üë`/`‚Üì`, type to filter,
`Enter` to drill down, `Esc` to go back.

The picker is a row-level action, not a sub-mode of the value editor:

- **`P` on an existing key row** ‚Äî selecting a field commits an
  `op://Vault/Item/field` reference straight into that key's pending
  value. No intermediate text modal.
- **`P` on a `+ Add` sentinel row** ‚Äî selecting a field stashes the
  reference, then opens an `EnvKey` modal so you can name the new key.
  Pressing `Enter` on the key name writes both at once.

The picker shows what is available to reference, never what's stored. Field
values are never read or displayed.

If you have multiple 1Password accounts signed in, the picker opens with an
Account pane first (`Enter` to drill into that account's vaults; `Esc` to
choose a different account). Single-account setups skip straight to the
vault list.

**Requirements:** 1Password CLI (`op`) must be installed and signed in.

- Not installed: `brew install 1password-cli` (macOS) or visit
  `1password.com/downloads/command-line/`, then run `op signin`.
- Not signed in: run `op signin` in your shell, then retry.
- No vaults visible: check Settings ‚Üí Developer ‚Üí CLI integration in the
  1Password app.

jackin' uses your existing `op` session ‚Äî there is no separate jackin' auth.
