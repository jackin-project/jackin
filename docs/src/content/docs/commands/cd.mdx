## Synopsis

```bash
jackin cd <CONTAINER> [DST]
```

Open a child shell whose working directory is set to a materialized isolated worktree path for `<CONTAINER>`. Use this to inspect, hand-edit, or run host-side git operations against a per-container worktree without joining the agent's container.

`jackin cd` does **not** mutate the parent shell process. When you `exit` the child shell, you return to your original shell and your original directory.

## Arguments

| Argument | Required | Description |
|---|---|---|
| `CONTAINER` | Yes | Container name (e.g. `jackin-the-architect`) or selector resolved to one. The container must have at least one record in `<data_dir>/jackin-<container>/.jackin/isolation.json`. |
| `DST` | No | Container destination path of a specific isolated mount (e.g. `/workspace/jackin`). Disambiguates when more than one isolated mount exists. |

## Mount selection

| Isolated mounts recorded | `DST` provided? | Behavior |
|---|---|---|
| Zero | — | Error — nothing to `cd` into. |
| One | Omitted | Use that mount. |
| One | Provided | Exact-match against the recorded `mount_dst`. Mismatch is an error. |
| More than one | Omitted | Interactive picker on a TTY. On non-interactive stdin, error and print the candidate destinations. |
| More than one | Provided | Exact-match against the recorded `mount_dst`. Mismatch is an error. |

The matching `dst` is matched literally — `/workspace/jackin` does not match `/workspace/jackin/`.

## Environment variables

The child shell inherits the operator's environment plus four jackin-specific vars set by `jackin cd`:

| Variable | Value |
|---|---|
| `JACKIN_CONTAINER` | The resolved container name. |
| `JACKIN_MOUNT_DST` | The container destination path of the selected isolated mount. |
| `JACKIN_ORIGINAL_SRC` | The host `src` recorded for this mount at materialization time. Useful for comparing the worktree against the original repo. |
| `JACKIN_WORKTREE` | The materialized worktree path on the host. Same as the child shell's working directory. |

## Exit codes

The child shell's exit code propagates back to the caller. `jackin cd` itself only adds non-zero exit codes for argument and resolution errors (no isolated mount found, ambiguous selection on non-TTY, container has no isolation records, etc.).

<Aside type="tip">
The child shell uses the real worktree path as `cwd` — jackin' does not spoof `PWD` or remap the original `src` location. That keeps `git status`, IDE workspace detection, and tooling that walks `..` honest.
</Aside>
