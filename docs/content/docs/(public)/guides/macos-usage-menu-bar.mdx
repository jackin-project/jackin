title: "macOS agent-usage menu bar"
description: "Install and use jackin❯ Desktop for host account quotas without an open Capsule."
---

**jackin❯ Desktop** shows **usage limits** (remaining/used quota %, reset countdowns, plan/status) for every jackin❯-supported agent surface — **without** requiring a running Capsule session. It does **not** show token unit prices or historical usage/spend trends.

On **macOS 26+** the glance popover, Usage sidebar/toolbar, and menu-bar chips use Apple **Liquid Glass** for navigation chrome only; usage cards and detail stay on standard materials so content stays legible (HIG). Older macOS uses system materials.

## What it shows

Only the seven providers jackin❯ supports: **Claude**, **Codex**, **Amp**, **Kimi**, **Grok**, **Z.AI / GLM**, and **MiniMax**. It never grows a CodexBar-style provider zoo (no Cursor, Gemini, Copilot, …).

Numbers come from the same Rust stack Capsule uses (`jackin-usage` + `jackin-protocol` views). Swift is display-only.

## Install

### Homebrew cask (stable)

Once a notarized release ships:

```bash
brew install --cask jackin-project/tap/jackin-desktop
```

That installs `JackinDesktop.app` from the immutable GitHub Release ZIP. Floor: **macOS 14 (Sonoma)+**. Uninstall: `brew uninstall --cask jackin-desktop` (removes the app; no root daemon or LaunchDaemon).

### Direct download

Download `jackin-desktop-<VERSION>-aarch64-apple-darwin.zip` plus `.sha256` / `.bundle` / `.sbom.json` from the [GitHub Releases](https://github.com/jackin-project/jackin/releases) page, verify with `cargo xtask release-verify <zip>`, then open the app. See [Verifying releases](/getting-started/verifying-releases/).

### Local developer build

```bash
mise install

# Build + verify + launch (one shot)
mise run desktop

# Or step by step — each step prints the absolute app path
mise run desktop-build -- 0.6.0 1   # ends with DESKTOP_APP=/…/JackinDesktop.app
mise run desktop-verify
mise run desktop-run                # launches the menu-bar app

# cargo equivalents
# cargo xtask desktop build --version 0.6.0 --build 1
# cargo xtask desktop verify
# cargo xtask desktop run
```

Default bundle: `native/dist/JackinDesktop.app`. The app is an **`LSUIElement`** process (no Dock icon — look at the menu bar). Quit: `osascript -e 'quit app "Jackin Desktop"'` or `killall JackinDesktop`. Uninstall: quit and delete the `.app` bundle. No root daemon is installed.

### Local notarization rehearsal (operators)

Requires a Developer ID Application identity and notarytool credentials (keychain profile or App Store Connect API key):

```bash
export DEVELOPER_ID_APPLICATION='Developer ID Application: Your Name (TEAMID)'
export NOTARY_PROFILE=jackin-notary
export JACKIN_APP_VERSION=0.6.0 JACKIN_APP_BUILD=1
mise run desktop-build -- 0.6.0 1
mise run desktop-sign-notarize
```

CI publishes the stapled ZIP from the **`release-macos`** environment on tag/`publish` mode (secret names only: see `native/README.md`).

## Credentials (host)

Probes read the **same host logins** operators already use for jackin❯ launch (home agent dirs and env keys first; container handoff paths only as last resort). See [ADR-011](/reference/adrs/adr-011-native-macos-usage-menu-bar/).

| Surface | Typical host roots |
|---|---|
| Claude | macOS Keychain (`Claude Code-credentials`, or a per-`CLAUDE_CONFIG_DIR` service for a custom config), then `~/.claude/.credentials.json`, `ANTHROPIC_*` |
| Codex | `~/.codex/auth.json` |
| Amp / Grok / Kimi | respective home secrets |
| Z.AI / MiniMax | `ZAI_API_KEY` / `MINIMAX_*` env |

No passwords are stored by the menu bar app. It reuses local sessions/keys already present for jackin❯.

Grok's plan/tier is read from xAI's billing service (the true subscription tier), not guessed from how you logged in, so a Free-tier browser login is labeled correctly; its prepaid credits and on-demand cap are shown as spending **limits**, never token prices or historical charts.

On a default macOS install Claude Code keeps its credential only in the login Keychain, so the first read may show a system consent prompt. Choosing **Allow** lets that read complete (macOS may ask again on a later refresh); **Always Allow** makes future reads silent. If you **Deny**, Claude stays disabled until you relaunch jackin❯ Desktop — no repeated prompt storm, and no other provider is affected. Over SSH or another headless session where the Keychain is unavailable, jackin❯ Desktop falls back to the credentials file exactly as before.

## Debug CLI (no UI)

```bash
jackin usage host snapshot --agent claude
jackin usage host snapshot --agent codex --format json
jackin usage host snapshot --agent claude --no-refresh
```

This drives `HostUsageRuntime` — the same view fields Capsule renders (`status_bar_label`, buckets, Money, honest `unavailable` / `refreshing`).

## Glance popover

Click the status item for a **CodexBar-style Overview** (clean-room layout):

1. **Provider tab grid** at the top — Overview plus every auto-detected provider in Capsule order. Enabled tabs show the Rust-owned glance percentage; severity underline/tint follows Rust.
2. **Overview or provider detail** under the grid — Overview shows one compact Rust-owned row per detected provider. Select a provider tab for its account chips, glance bar, quota-window bars, reset/pace lines, and allowed quota-bound credits.
3. Click an agent header to open the **Usage window** focused on that provider; the popover closes as it opens.

Footer: a single **Refresh** (⌘R) row plus the next-refresh caption. When no provider is detected, the preview tells you to sign in to a supported agent; there is no credential-harvesting or Settings UI.

**Status-item clicks.** **Left-click** a provider item toggles the popover on that provider's tab. **Right-click** any provider item opens a static context menu with exactly three rows — **Open Usage Window**, **Refresh**, **Quit jackin❯ Desktop** — and nothing else. Open Usage Window opens/focuses the Usage window on its current selection (Overview when none); Refresh runs the same forced Rust-owned refresh the popover footer does; Quit terminates the app.

## Usage window

Click an agent identity header in the popover, or choose **Open Usage Window** from a provider item's right-click menu. A normal resizable window opens:

- **Sidebar** — Overview plus every detected provider in the Capsule dialog's tab order, each with a severity accent.
- **Overview** — one row per provider showing the same identity, plan, headline, and reset strings the glance popover uses. When no credentials are detected the list shows the hint *no agent credentials found*.
- **Detail** — selecting a provider shows the **full Capsule-parity card**: the exact same fields, strings, and order the Capsule usage dialog renders for that account and fetch (identity, plan, auth origin, every quota window with its meter, pace, run-out, reset, and money caps). Rust produces the whole card; the window draws it row for row and invents nothing. Multi-account providers show account chips that drive the same selection as the popover.
- **⌘R** refreshes; **Esc** closes. The app stays a status-item process (`LSUIElement` — no Dock icon).

All numbers and labels are the same Rust views the Capsule usage dialog uses. Rust models each provider's semantic Session/Weekly glance slots (Weekly-only where a provider has no Session window), while **Amp Free** uses its exact **Daily** allowance (`N% remaining today`, resets daily). Individual and per-workspace credit balances stay detail-only quota bounds and never become the status headline.

## Multi-account

jackin❯ Desktop lists **every known account** per provider — the current host login, accounts recorded in the durable menu-bar store after prior logins, and shared usage snapshots written by capsules for other OAuth identities (same `account_key` coordination jackin❯ uses host-wide).

- **Glance + Usage detail** show account **pills** when more than one account is known; selecting a pill switches the card snapshot (selection is persisted).
- **Status bar** still shows provider remaining for the selected/live account; it never invents a second provider chip from empty data.
- Probing still uses host credential roots for the live login; additional accounts appear when jackin❯ has already stored or shared their usage views (same model as multi-workspace / multi-login coordination).

## Menu bar appearance

jackin❯ Desktop creates **one native status item per auto-detected provider** in Rust's Capsule order. Each item shows the provider's template icon plus one selected-account glance percentage: **Weekly remaining** for Codex, Claude, Grok, Z.AI, Kimi, and MiniMax; **Daily remaining** for Amp Free. Providers without numeric data yet show the icon with `—`; stale or unavailable last-known data stays visible and dimmed. When no provider is detected, one jackin❯ fallback item keeps the preview reachable.

There is no Settings surface in this release. Provider membership is automatic, the Rust-selected glance label is fixed, and refresh remains Rust-owned with its cooldown and minimum cadence.

## Privacy

- Local process; probes use your host credentials and provider APIs the same way Capsule does.
- Refreshing usage in the menu bar updates the same account snapshot every jackin❯ container reads (and vice versa), so numbers always match across surfaces.
- No second OAuth client matrix in Swift.
- Telemetry follows existing jackin❯ policy when the host binary enables it; the menu bar does not invent a parallel sink.
