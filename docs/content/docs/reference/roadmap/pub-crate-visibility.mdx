**Status**: Partially implemented — lint enabled, low-hanging-fruit items fixed (2026-06-02); ~100 warnings remain for the second pass.

## Problem

`jackin` has a real library surface: <RepoFile path="src/lib.rs" /> publicly exports most top-level modules and <RepoFile path="src/bin/role.rs" /> consumes the library. That means a blanket "binary crate ⇒ convert everything to `pub(crate)`" pass would be wrong. The actual readability problem is narrower: some items are still wider than needed.

## What shipped (2026-06-02)

- **Lint enabled**: `unreachable_pub = "warn"` added to `[lints.rust]` in <RepoFile path="Cargo.toml" />. The warning fires on every `cargo check`, making the advisory guidance visible without blocking builds.
- **Low-hanging-fruit items fixed** (all 8 roadmap-specified items, with corrected names):
  - <RepoFile path="src/workspace/planner.rs" /> — `WorkspaceCreatePlan`, `WorkspaceEditPlan`, `plan_create`, `plan_edit`, `apply_isolation_overrides` → `pub(crate)`
  - <RepoFile path="src/config/mounts.rs" /> — `MountEntry` → `pub(crate)`
  - <RepoFile path="src/config/roles.rs" /> — `BUILTIN_ROLES` (name changed from `BUILTIN_AGENTS`) → `pub(super)`
  - <RepoFile path="src/runtime/launch.rs" /> — `inspect_attach_outcome`, `EnvLayerState`, `LaunchError`, `verify_credential_env_present` → `pub(super)`
  - <RepoFile path="src/runtime/attach.rs" /> — `JACKIN_STATUS_CMD`, `parse_session_count`, `docker_unavailable_msg` → `pub(crate)` (isolation/finalize.rs and app/mod.rs access these from outside runtime; pub(super) would break those paths)

## Remaining work

~100 warnings remain across 26 files (run `cargo check 2>&1 | grep "unreachable_pub"` for the current set). The largest clusters are in private `console/` sub-modules (`domain`, `services/`) and `runtime/universe.rs`. These require deciding the intended visibility per-item before bulk-converting.

The next pass should:
1. Decide the intended library surface in <RepoFile path="src/lib.rs" /> — what truly needs to stay `pub` for `main.rs`, `jackin-role`, and tests.
2. Work through the private-module clusters (console services, runtime universe) where `pub(super)` is the obvious narrowing.
3. Verify `cargo nextest run` still passes after each batch.

## What stays `pub`

- `pub fn run` and other intentionally exported library entrypoints used by sibling binaries
- validation helpers consumed by <RepoFile path="src/bin/role.rs" />
- any types that are deliberately part of the internal library boundary exposed from <RepoFile path="src/lib.rs" />

## Caveats

Do not treat `unreachable_pub` as a turnkey "convert hundreds of bare `pub` items" script. It is a cleanup aid for the current exported shape, not a substitute for deciding what the library boundary should be.
