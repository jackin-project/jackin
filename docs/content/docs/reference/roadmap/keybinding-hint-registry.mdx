**Status**: Resolved

The core `Keymap<A>` registry shipped in full across the console, launch, and shared widget layer (original keybinding-hint-registry Phases 0–3), and launch Ctrl+C / Ctrl+Q cancellation shipped across all seven phases. The four Capsule hint/dispatch gaps (items A–D), the pane-resize keymap registration (item E), the enforcement audit (item F), and the twelve test module layout violations (items G.1–G.12) have all landed; the Debug-info dialog hint bar was additionally unified onto the shared `debug_info_hint_spans` builder so the console and launch surfaces render byte-identical hints. The complete implementation record is preserved below as a checkbox-driven account of what changed and how each item was verified.

## What shipped

### Keymap registry (Phases 0–3)

`Keymap<A>` in <RepoFile path="crates/jackin-tui/src/keymap.rs" /> is the single source of truth for keybinding dispatch and hint advertisement. `Keymap::dispatch(chord)` and `Keymap::hint_spans()` both read from the same `bindings: &'static [KeyBinding<A>]` table, making divergence architecturally impossible for `Shown` and `HiddenAlias` bindings. The full migration covered: all five shared-widget keymaps (`ConfirmState`, `TextInputState`, `SelectListState`, `ErrorPopupState`, `SaveDiscardState`); the capsule `PREFIX_COMMAND_KEYMAP` and dialog keymaps (`CONFIRM_KEYMAP`, `FILTER_LIST_KEYMAP`, `RENAME_KEYMAP`, `READ_ONLY_DISMISS_KEYMAP`); the console keymaps (`WORKSPACE_LIST_KEYMAP`, `PREVIEW_PANE_KEYMAP`, all editor/settings tab keymaps, and all row-level `Keymap<()>` display keymaps) in <RepoFile path="crates/jackin-console/src/tui/keymap.rs" />; and the launch keymaps (`COCKPIT_KEYMAP`, `BUILD_LOG_KEYMAP`, `FAILURE_KEYMAP`, `CONTAINER_INFO_KEYMAP`) in <RepoFile path="crates/jackin-launch/src/tui/keymap.rs" />. The lookbook migrated to `SIDEBAR_KEYMAP` and `PREVIEW_KEYMAP`. Dead hint constants (`SELECT_CANCEL_HINT`, `DISMISS_HINT`, `DISMISS_CANCEL_HINT`, `CONFIRM_DISMISS_HINT`) were removed. All 43 operational screens have a rendered hint bar. Dispatch-layer tests cover every keymap. `raw_bytes_to_chord` in <RepoFile path="crates/jackin-tui/src/keymap.rs" /> bridges the capsule's PTY byte stream to `KeyChord`. Hint-row graceful truncation (`truncate_spans_to_cols`) shipped for narrow terminals in <RepoFile path="crates/jackin-capsule/src/tui/components/chrome.rs" />.

### Launch exit keybindings (Phases 1–7)

`Ctrl+C` fires an immediate hard cancel through a `CancellationToken` in <RepoFile path="crates/jackin-launch/src/progress.rs" />, `while_waiting()` `tokio::select!`, and `RichRenderer::restore_terminal()` called inside the render task before any pipeline future resumes — guaranteeing terminal restoration before cleanup output. `Ctrl+Q` opens the shared `exit_confirm_state()` dialog from <RepoFile path="crates/jackin-tui/src/components/confirm_dialog.rs" /> on all three surfaces (console, launch cockpit, capsule). `Ctrl+C` wins over an open dialog. A `LaunchCancelled` sentinel exits `0` with no error line. `kill_on_drop(true)` on `ShellRunner` children ensures `docker build` is killed on cancel so cleanup never blocks behind a still-running build process. All pipeline stages that can block for more than a second go through `while_waiting()`. The TUI chrome design — hint bar, not status bar — is documented in [TUI chrome reference](/reference/tui/chrome).

## Implementation record (all items landed)

This section is the implementation plan as executed. Each item names the exact file and location that changed, the exact fix applied, and the acceptance criterion — every box is now checked. Items A–D shipped as one PR, item E as a separate PR, and items G.1–G.12 were batched across PRs.

**Universal enforcement rule (applies to all surfaces, all future PRs):**

> Every hint bar must be constructed by calling `.hint_spans()` or `.glyph_for()` on the `Keymap<A>` that owns dispatch for that surface. No `HintSpan::Key(…)` literal may appear in a hint builder unless it represents a genuinely unregisterable input (CSI multi-byte sequence, mouse event, or dynamic runtime value). Every such literal must carry a `// UNREGISTERABLE(<reason>)` comment naming the specific reason. Any `HintSpan::Key(…)` without a keymap entry and without an `// UNREGISTERABLE` comment is a merge blocker.

### A — Capsule: Ctrl+Q must dispatch through `CAPSULE_GLOBAL_KEYMAP`

**Why it is wrong today.** <RepoFile path="crates/jackin-capsule/src/tui/input.rs" /> line 351 dispatches Ctrl+Q as a raw byte comparison (`else if b == CTRL_Q { events.push(InputEvent::RequestExit); }`) where `CTRL_Q: u8 = 0x11` is defined at line 533. This bypasses the `Keymap<A>` registry entirely. Three locations in <RepoFile path="crates/jackin-capsule/src/tui/components/dialog/hint.rs" /> (lines 30, 45, 62) hardcode `HintSpan::Key("Ctrl-Q")` manually instead of deriving the span from the keymap. If the key ever changes or the binding is updated, both dispatch and every hint must be updated by hand — the guarantee the registry provides does not apply here.

**Files to change.**

- <RepoFile path="crates/jackin-capsule/src/tui/keymap.rs" /> — add `GlobalCapsuleAction` enum and `CAPSULE_GLOBAL_KEYMAP` static
- <RepoFile path="crates/jackin-capsule/src/tui/input.rs" /> — remove `const CTRL_Q: u8 = 0x11` and the raw `else if b == CTRL_Q` arm; add dispatch through `CAPSULE_GLOBAL_KEYMAP`
- <RepoFile path="crates/jackin-capsule/src/tui/components/dialog/hint.rs" /> — remove the three `HintSpan::Key("Ctrl-Q")` literals; in every hint builder function, append `CAPSULE_GLOBAL_KEYMAP.hint_spans()`

**Exact code for `keymap.rs`.**

```rust
pub enum GlobalCapsuleAction {
    RequestExit,
}

pub static CAPSULE_GLOBAL_KEYMAP: Keymap<GlobalCapsuleAction> = Keymap::new(&[
    KeyBinding {
        chords:     &[KeyChord::ctrl(LogicalKey::Char('q'))],
        action:     GlobalCapsuleAction::RequestExit,
        hint:       Some("quit"),
        visibility: Visibility::Shown,
        glyph:      None,  // auto-derives "Ctrl-Q"
    },
]);
```

**Exact change for `input.rs`.**

Remove line 533: `const CTRL_Q: u8 = 0x11;`

Remove the arm at line 351 (exact text): `else if b == CTRL_Q { events.push(InputEvent::RequestExit); }`

Before the first `if Some(b) == self.prefix_key` check or equivalent outer dispatch, insert:

```rust
if let Some(action) = CAPSULE_GLOBAL_KEYMAP.dispatch(KeyChord::from_byte(b)) {
    match action {
        GlobalCapsuleAction::RequestExit => events.push(InputEvent::RequestExit),
    }
    return events;
}
```

**Exact change for `hint.rs`.**

1. Remove `HintSpan::Key("Ctrl-Q")` at line 30 (prefix-awaiting branch), line 45 (scrollback branch), and line 62 (normal-mode branch), along with any adjacent `HintSpan::Text("quit")` and `HintSpan::GroupSep` that were manually paired with them.
2. In every public hint builder function in the file (`main_view_hint`, `scroll_hint_spans` wrappers, any others), before `return spans;`, add: `spans.extend(CAPSULE_GLOBAL_KEYMAP.hint_spans());`

**Acceptance criteria.**

- [x] `CAPSULE_GLOBAL_KEYMAP` exists in `keymap.rs` with one `Shown` binding for `Ctrl+Q → RequestExit`.
- [x] `const CTRL_Q` and the raw else-if arm are deleted from `input.rs`.
- [x] Dispatch goes through `CAPSULE_GLOBAL_KEYMAP.dispatch(KeyChord::from_byte(b))`.
- [x] No `HintSpan::Key("Ctrl-Q")` literal exists anywhere in `hint.rs`.
- [x] Every hint builder in `hint.rs` calls `CAPSULE_GLOBAL_KEYMAP.hint_spans()` to produce the Ctrl-Q span.
- [x] `cargo test -p jackin-capsule` passes.
- [x] Unit test in `crates/jackin-capsule/src/tui/keymap/tests.rs` asserts `CAPSULE_GLOBAL_KEYMAP.dispatch(KeyChord::from_byte(0x11))` returns `Some(GlobalCapsuleAction::RequestExit)`.
