**Status**: Open — active program. The TUI surfaces are extracted and restructured; the remaining work is concentrated in the root `src/` binary crate. Re-baked 2026-06-02.

## Goal

Make AI-generated code verifiable and maintainable. The codebase is ~170K lines of production Rust across a 7-crate workspace plus a root binary crate, built with heavy AI assistance. The recurring failure mode is the same one this program exists to fix: files outgrow their module boundaries, the same pattern gets copy-pasted instead of shared, and single functions grow past the point where a reviewer can hold them in their head.

The TUI half of that problem is now largely solved — see [What shipped](#what-shipped). **The unsolved half lives in the root `src/` crate**: the runtime/launch pipeline, the app dispatch loop, instance auth provisioning, and operator-env resolution are still monolithic, duplicated, and undocumented. This page is the plan to bring `src/` up to the standard the extracted crates already meet.

## What shipped

These items are done. They are listed here so the remaining work is not confused with work already complete.

- **TUI architecture & surface extraction.** All terminal UI moved into a 7-crate workspace (`jackin-tui`, `jackin-console`, `jackin-launch`, `jackin-capsule`, `jackin-protocol`, `jackin-tui-lookbook`, `jackin-build-meta`) following one Elm Architecture boundary. Design rules live in [TUI Design](/reference/tui/). This eliminated the single largest duplication cluster the original audit flagged (the `console/tui/view/` render layer).
- **Architecture Decision Records.** Four ADRs written and published at [/reference/adrs/](/reference/adrs/) (single-crate vs workspace, toolchain, ratatui, capsule pane rendering).
- **Snapshot-test net started.** Four `insta` snapshot tests cover the largest composite panes (list, settings, editor-general, editor-mounts) in <RepoFile path="src/console/tui/view/snapshot_tests.rs" />. See [Snapshot tests for TUI](/reference/roadmap/snapshot-tests-tui/) for the remaining coverage.
- **`pub(crate)` visibility pass progressing.** Root-crate `pub(crate) fn` share rose from ~11% to ~31% as the surface crates drew explicit public boundaries. Tracked in [pub(crate) visibility](/reference/roadmap/pub-crate-visibility/).
- **MSRV & toolchain pin.** `rust-toolchain.toml` enforced in CI, MSRV floor verified (ADR-002).
- **Shared TUI scrollbar primitive.** Metrics, clamping, drag/click mapping, and capsule scrollback unified in `jackin_tui::scroll`; contract documented under [TUI Design → Navigation](/reference/tui/navigation/).
- **`PROJECT_STRUCTURE.md` refresh.** Root map delegates detailed source navigation to the [Codebase Map](/reference/getting-oriented/codebase-map/).

## What remains

The remaining work is four tracks. Do them in roughly this order — DRY extraction and the doc/spec net make the file splits smaller and safer, and the file splits must precede the deeper crate split.

### 1. DRY extractions (do first — lowest risk, highest reach)

Every extraction below has 3+ real call sites. No speculative abstractions.

**Shared string constants.** None of these are extracted yet; all remain inline literals.

| Constant | Literal | Occurrences |
|---|---|---|
| `MANIFEST_FILENAME` | `"jackin.role.toml"` | 160 |
| `DOCKERFILE_NAME` | `"Dockerfile"` | 235 |
| `BASE_DOCKERFILE_FROM` | `"FROM projectjackin/construct:trixie\n"` and variants | ~100 |

**Named style constants.** The color palette is centralized (`jackin_tui::theme`), but the `Style::default().fg(…).add_modifier(BOLD)` boilerplate is rebuilt 235+ times across the TUI crates. Extract `BOLD_WHITE`, `DIM`, `GREEN`, `BORDER`, `DANGER` into `jackin-tui` so callers reference a style, not a construction chain.

**Auth provisioning.** <RepoFile path="src/instance/auth.rs" /> (3,302 L) holds six near-identical `provision_*_auth` functions (Codex, GitHub, Claude, Amp, Kimi, OpenCode). The Sync arm (read host file → write to role-state → `HostMissing` on `NotFound`) and the wipe-then-return for `ApiKey`/`Ignore` are copy-pasted across them. Extract `provision_file_credential(target, host_path, mode)`.

**Agent match-arm dispatch.** Adding a new agent currently requires editing eight files that each carry an `Agent::Claude => …, Agent::Codex => …` arm: <RepoFile path="src/config/roles.rs" />, <RepoFile path="src/instance/mod.rs" />, <RepoFile path="src/runtime/launch.rs" />, <RepoFile path="src/runtime/image.rs" />, <RepoFile path="src/derived_image.rs" />, <RepoFile path="src/agent_binary.rs" />, <RepoFile path="src/version_check.rs" />, and <RepoFile path="src/manifest/validate.rs" />. Replace with trait dispatch or a single dispatch macro.

**Docker resource derivation.** 40+ inline `format!("{name}-dind")` / `format!("{name}-net")` derivations and a duplicated DinD-sidecar `docker run` arg list across <RepoFile path="src/runtime/launch.rs" />, <RepoFile path="src/runtime/attach.rs" />, and <RepoFile path="src/runtime/cleanup.rs" />. Extract a resource-name struct and a `run_dind_sidecar` helper.

**Test infrastructure.** Consolidate the repeated `FakeRunner` definitions, `seed_minimal_role_repo` blocks, `test_workspace` / `simple_mount` constructors, and `config_with_agents` helpers into a shared `test_support` module.

### 2. Reduce mega-functions

53 functions in `src/` carry `#[allow(clippy::too_many_lines)]` — up from 35 at the original audit, so this is regressing, not improving. The worst offenders by suppression count are <RepoFile path="src/console/tui/input/global_mounts.rs" />, <RepoFile path="src/runtime/launch.rs" />, <RepoFile path="src/console/tui/input/save.rs" />, and <RepoFile path="src/console/tui/input/mouse.rs" />. The two largest single functions are `fn run` in <RepoFile path="src/app/mod.rs" /> (1,406 L, 51-arm match) and `load_role_with` in <RepoFile path="src/runtime/launch.rs" /> (1,003 L). Each `too_many_lines` suppression added in a new PR is a regression on this track; reviewers should treat a new suppression as a request to extract, not annotate.

### 3. Split the root-crate monoliths

Snapshot/behavioral coverage and the DRY pass above must land on a file before it is split. The four files with dedicated design pages:

- [Split `runtime/launch.rs`](/reference/roadmap/split-runtime-launch/) — now **9,482 L** (was 7,966), the hardest file in the codebase; behavioral spec required first.
- [Split `console/tui/input/editor.rs`](/reference/roadmap/split-input-editor/) — **4,212 L**, one module per editor tab.
- [Split `operator_env.rs`](/reference/roadmap/split-operator-env/) — **4,209 L**, separate the shared API, CLI client, env resolution, diagnostics, and picker metadata.
- [Split `app/mod.rs`](/reference/roadmap/split-app-mod/) — **3,966 L**, separate dispatch from the workspace/config command handlers.

Other root-crate files over 2,000 L that need splitting (no dedicated page yet): <RepoFile path="src/instance/auth.rs" /> (3,302, after the auth-provisioning DRY pass), <RepoFile path="src/console/tui/input/global_mounts.rs" /> (2,871), <RepoFile path="src/console/tui/state.rs" /> (2,707), <RepoFile path="src/console/tui/input/mouse.rs" /> (2,676), <RepoFile path="src/isolation/materialize.rs" /> (2,441), <RepoFile path="src/config/editor.rs" /> (2,434), <RepoFile path="src/workspace/token_setup.rs" /> (2,388), and <RepoFile path="src/isolation/finalize.rs" /> (2,134).

Three crate-side files also exceed 2,000 L and should be split, though they are lower priority than the root-crate work: <RepoFile path="crates/jackin-capsule/src/daemon.rs" /> (3,901 L, but ~2,900 of that is one `#[cfg(test)]` module — split the test suite into per-submodule files first), <RepoFile path="crates/jackin-capsule/src/tui/components/dialog.rs" /> (3,207), and <RepoFile path="crates/jackin-console/src/tui/components/op_picker.rs" /> (2,877).

### 4. Module contracts & behavioral specs

Six `src/` modules have **zero** `//!` orientation docs: `isolation/`, `instance/`, `app/`, `manifest/`, `agent/`, and `tui/`. These are also the modules with the largest undocumented files (`auth.rs`, `materialize.rs`, `app/mod.rs`). Prioritize them. Tracked across:

- [Module contracts](/reference/roadmap/module-contracts/) — `//!` docs for the 79 files still missing them.
- [Behavioral spec: runtime/launch.rs](/reference/roadmap/behavioral-spec-runtime-launch/) — verified invariants for the `jackin load` critical path; gates the launch split.
- [Behavioral spec: op_picker](/reference/roadmap/behavioral-spec-op-picker/) — invariants for the 1Password picker state machine.
- [Per-directory README](/reference/roadmap/per-directory-readme/) and [Developer Reference setup](/reference/roadmap/developer-reference-setup/) — AI-native orientation files and the browsable internals section.
- [CI gate: PROJECT_STRUCTURE.md](/reference/roadmap/ci-project-structure-gate/), [cc-sdd workflow](/reference/roadmap/agent-workflow-cc-sdd/), and [Publish CONTRIBUTING + TESTING](/reference/roadmap/move-contributing-testing/) — policy and workflow follow-through.

## Deferred — deeper crate split

The 2026 workspace extraction was **surface-oriented**: it carved out the TUI crates but left the entire non-visual core (runtime, config, instance, isolation, app, manifest, cli) inside the root `src/` binary crate. The [Cargo workspace split](/reference/roadmap/cargo-workspace-split/) item describes the tier-based split that would extract `jackin-core`, `jackin-runtime`, and `jackin-config`. Its original LOC trigger (150 K) is now met, but executing it depends on the file splits in track 3 first — the monoliths must become focused modules before they can become focused crates. The one cross-module dependency that previously blocked it (console → runtime) now lives correctly inside a service adapter at <RepoFile path="src/console/services/role_load.rs" />, so the boundary is cleaner than at the original audit.

The [rustdoc JSON → Fumadocs pipeline](/reference/roadmap/rustdoc-json-fumadocs/) remains deferred until `//!` coverage is high enough to be worth publishing.
