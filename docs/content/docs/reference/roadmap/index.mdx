## Current status

jackin' is a functional proof of concept, not a stable product line. **`Claude Code`, `Codex`, `Amp`, `Kimi`, and `OpenCode` ship today as built-in agent runtimes**, with namespaced roles, workspace management, per-mount isolation for parallel agents, layered authentication forwarding (including `1Password` references), and an interactive operator console (TUI). The roadmap below is expected to change aggressively as the project learns which concepts deserve to survive.

### Security improvements

- **[Network egress policy](/reference/roadmap/network-egress-policy/)** ‚Äî outbound policy per role/workspace, with explicit enforcement-quality reporting across DinD, future microVM backends, SSH remotes, and Kubernetes. Inspired by Docker Sandboxes' host-side proxy model, Hazmat's network tiering, and the upstream `anthropics/claude-code` devcontainer allowlist approach (status: open ‚Äî design proposal)
- **[Process-level sandboxing](/reference/roadmap/process-level-sandboxing/)** ‚Äî per-operation isolation using OS-native mechanisms (Bubblewrap + Seccomp + Landlock on Linux, Apple Seatbelt on macOS) to restrict individual agent tool calls to only the paths, network hosts, and environment variables they need. Informed by [Zerobox](https://github.com/afshinm/zerobox) (the open-source library powering OpenAI Codex CLI's sandbox mode) and the three-tier isolation model: process sandbox ‚Üí container ‚Üí microVM (status: proposed ‚Äî research captured, no implementation committed)
- **Credential proxy** ‚Äî proxy-based credential injection to avoid storing tokens inside containers
- **[Agent version pinning](/reference/roadmap/reproducibility-pinning/)** ‚Äî pin role repos to tagged versions for reproducible builds, with explicit `--update` to advance (status: open ‚Äî design proposal)
- **[Auth health and operator visibility](/reference/roadmap/auth-health-and-visibility/)** ‚Äî pre-launch credential validity probe, `jackin auth status` command, console auth health indicators, and launch-summary diagnostics so auth failures surface before and not during a session (status: open ‚Äî design proposal)
- **[Reliable Claude authentication strategy](/reference/roadmap/claude-auth-strategy/)** ‚Äî move beyond copied session auth toward clearer, more durable modes for long-lived and concurrent agent runtimes (status: deferred ‚Äî needs design work)
- **[Docker runtime hardening contract](/reference/roadmap/docker-runtime-hardening-contract/)** ‚Äî keep Docker as the default runtime while making its security posture explicit: profiles, DinD hardening, resource budgets, read-only root evaluation, network-policy defaults, and launch-contract reporting (status: open ‚Äî design proposal)
- **[OrbStack isolated machine backend](/reference/roadmap/orbstack-isolated-machine-backend/)** ‚Äî macOS backend research for running jackin' role containers and private Docker workflows inside OrbStack isolated machines, with explicit shares and honest shared-kernel risk language (status: open ‚Äî research and design proposal)
- **[smolvm backend research](/reference/roadmap/smolvm-backend-research/)** ‚Äî open-source Rust/libkrun VM backend research for true VM-per-workload isolation, OCI-image execution, network-off defaults, allow-host egress, possible future SDK embedding, and a direct comparison with OrbStack isolated machines and Docker Sandboxes (status: open ‚Äî research and design proposal)
- **[Selectable sandbox backends](/reference/roadmap/selectable-sandbox-backends/)** ‚Äî umbrella for Docker, OrbStack isolated machines, Kubernetes, and later true microVM providers; detailed backend work now lives in focused roadmap items (status: deferred ‚Äî umbrella design)
- **[Devcontainer parity](/reference/roadmap/devcontainer-parity/)** ‚Äî comparison with the upstream `anthropics/claude-code` devcontainer setup; informs network policy + reproducibility decisions (status: proposed ‚Äî research captured, no implementation committed)

### Reactive daemon program

- **[jackin' daemon ‚Äî umbrella](/reference/roadmap/jackin-daemon/)** ‚Äî introduce the long-running per-operator-user host process jackin' will use for reactive features. One umbrella item that decides lifecycle, install method, control socket, security posture, and log redaction once so each reactive feature plugs into one daemon shape. - **[jackin' Desktop Agent Hub](/reference/roadmap/jackin-desktop-agent-hub/)** ‚Äî native macOS menu bar and desktop companion for active jackin' workspaces, isolated agent sessions, PR jump links, and built-in-runtime account state (Claude, Codex, Amp, Kimi, OpenCode), with CodexBar and OpenUsage as scoped references for account quota surfaces. Keeps the agent TUIs as the primary agent UI while using the daemon as the shared state/event backend (status: open ‚Äî design proposal)
- **[Live bidirectional auth sync](/reference/roadmap/live-auth-sync/)** ‚Äî Phase 2 adapter. Keep host and every running container in lock-step on each auth axis (`gh`, Claude, Codex, Amp, ‚Ä¶). Subsumes the launch-time `sync` mode's bidirectional follow-up; reconsiders the `sync` name in the process (status: open ‚Äî design proposal)
- **[Agent runtime status authority](/reference/roadmap/agent-runtime-status/)** ‚Äî Phase 2 state source. Herdr-class in-container authority for `working`, `blocked`, derived `done`, `idle`, `unknown`, and stuck diagnostics, informed by multicode, CCManager, Agent Session Manager, WezTerm Agent Deck, ccmux, TUICommander, Codemux, and tmux-agent-status. Uses semantic runtime hooks/APIs, foreground-process ownership, visible-screen signals, shell markers, and cursor/readiness probes instead of treating PTY silence as attention-needed (status: open ‚Äî design proposal)
- **[Agent attention prompts](/reference/roadmap/agent-attention-prompts/)** ‚Äî Phase 2 adapter. Host-side OS notifications when an agent inside a Capsule-managed role container is waiting on operator input or has finished unseen work. Consumes the agent runtime status authority, escalates from silent toast to sound after a configurable timeout, and focuses the right terminal tab where supported. Targets the biggest operator throughput drag: idle wall-clock waiting on agents that don't surface their waiting state (status: open ‚Äî design proposal)
- **[Host bridge ‚Äî secrets and approved host actions](/reference/roadmap/host-bridge/)** ‚Äî Phase 3 adapter. Operator-mediated channel for agents to request a single secret value or invoke a single host command without tearing down the container. Auto-registered MCP server, TouchID / polkit / password approval per request, per-workspace allowlist / blocklist / disabled policies, audit log. Closes the *"agent needs one thing from the host, mid-session"* gap that today forces full-restart workarounds (status: open ‚Äî design proposal)
- **[Container credential exposure ‚Äî beyond env injection](/reference/roadmap/container-credential-exposure/)** ‚Äî Phase 3 follow-on. Auth tokens and operator env values resolved from `op://` references become agent-readable when exported into the container today. Captures the trajectory from documented exposure to file-mount compatibility, command-scoped secret handles, Docker Sandboxes-style credential proxying, and daemon-mediated `secret.request` / `secret.run` grants where the agent gets a handle rather than the raw value (status: open ‚Äî design proposal)

Default dangerous-mode behavior preserved; role authors can override or extend (status: open ‚Äî design proposal)
- **[Public attribution and project growth](/reference/roadmap/public-attribution-and-growth/)** ‚Äî research how jackin' can use visible software-development surfaces, especially Git commit trailers and PR metadata, to make the project discoverable without silently mutating operator repositories or misusing authorship semantics (status: open ‚Äî research and design proposal)

### Codebase health

- **[Workspace-native PR verification](/reference/roadmap/pr-verification/)** ‚Äî a console flow that builds the per-PR Verify-locally loop into the console: select an instance that's working on a pull request, see its PR details, switch the sidebar into review mode, and run the build/test/serve commands the PR's changes call for ‚Äî host-side against an isolated checkout, so jackin' can verify its own PRs without touching the operator's real setup or live instance (status: open ‚Äî design proposal)
- **[Test infrastructure & behavioral specs](/reference/roadmap/test-infra-behavioral-specs/)** ‚Äî extract shared test helpers into a `jackin-test-support` crate, split mega test files, wire snapshot testing, write behavioral specs for all major components (capsule daemon, operator console, launch pipeline, auth forwarding), and add `proptest` + `cargo-fuzz` for parser surfaces (manifest validation, env resolver, migration chain) (status: open ‚Äî design proposal)
- **[Visual snapshot testing (CLI & TUI)](/reference/roadmap/visual-snapshot-testing/)** ‚Äî canonical home for visual / styled-snapshot regression testing of every rendered surface (operator console, capsule, launch TUI, and plain CLI output). Capture the full styled cell (glyph + colours + every modifier), store deterministic SVG goldens that GitHub renders inline so changes show as a before/after diff in the pull request, and keep the whole pipeline Rust-only (extend the first-party `buffer_to_svg` encoder for TUI; evaluate `term-transcript` for CLI output). Absorbs the former *Snapshot tests for TUI render* item; a thin text-only `insta` net and the component SVG lookbook have shipped (status: open ‚Äî research and design proposal)
- **[CI matrix split](/reference/roadmap/ci-matrix-split/)** ‚Äî split the monolithic `check` CI job into lint, unit, integration, e2e-docker, and capsule jobs for faster feedback and clearer failure attribution (status: open ‚Äî design proposal)
- **[Rust CI tooling and dependency hygiene](/reference/roadmap/rust-ci-tooling/)** ‚Äî umbrella for Rust CI layers beyond fmt/clippy/nextest/MSRV. Dependency hygiene has shipped with `cargo-audit`, `cargo-deny`, and `cargo-shear`; Codebook spell-check now runs as two whole-branch docs/source jobs through `codebook-lsp lint`; coverage and snapshot expansion remain open. Phase 2 (at/after first release) keeps `dist` for the Homebrew-tap + `cargo-zigbuild` release pipeline and `git-cliff` for changelogs. Includes a verified license audit (zero GPL/AGPL today ‚Äî the ban is a preventive guard) and records why `release-plz`, `cargo-semver-checks`/`cargo-public-api`, `cargo-vet`/`cargo-crev`, `cargo-mutants`, and `cargo-geiger` were skipped or deferred for a binary CLI workspace (status: partially implemented)
- **[Open review findings catalog](/reference/roadmap/open-review-findings/)** ‚Äî living backlog of review findings with accepted-exception annotations. Code review and automated scanning consult this catalog (see <RepoFile path="AGENTS.md" /> ‚Üí "Code review & automated scanning").

### Documentation infrastructure

- **[Markdown linting for docs](/reference/roadmap/docs-markdown-linting/)** ‚Äî uniform Markdown lint pipeline for the docs site (status: proposed ‚Äî design captured, no implementation committed)
- **[Move documentation to a separate repository](/reference/roadmap/docs-separate-repository/)** ‚Äî promote `docs/` out of the CLI repo into its own `jackin-project/jackin-docs` repo so the ecosystem-level docs have an ecosystem-level home; the load-bearing question is how to keep the docs as fresh as same-repo co-location makes them today (status: deferred ‚Äî needs design pass on the cross-repo freshness story)
