## Current status

jackin' is a functional proof of concept, not a stable product line. **`Claude Code`, `Codex`, `Amp`, `Kimi`, and `OpenCode` ship today as built-in agent runtimes**, with namespaced roles, workspace management, per-mount isolation for parallel agents, layered authentication forwarding (including `1Password` references), and an interactive operator console (TUI). The roadmap below is expected to change aggressively as the project learns which concepts deserve to survive.

### Security improvements

- **[Network egress policy](/reference/roadmap/network-egress-policy/)** — outbound policy per role/workspace, with explicit enforcement-quality reporting across DinD, future microVM backends, SSH remotes, and Kubernetes. Inspired by Docker Sandboxes' host-side proxy model, Hazmat's network tiering, and the upstream `anthropics/claude-code` devcontainer allowlist approach (status: open — design proposal)
- **[Process-level sandboxing](/reference/roadmap/process-level-sandboxing/)** — per-operation isolation using OS-native mechanisms (Bubblewrap + Seccomp + Landlock on Linux, Apple Seatbelt on macOS) to restrict individual agent tool calls to only the paths, network hosts, and environment variables they need. Informed by [Zerobox](https://github.com/afshinm/zerobox) (the open-source library powering OpenAI Codex CLI's sandbox mode) and the three-tier isolation model: process sandbox → container → microVM (status: proposed — research captured, no implementation committed)
- **Credential proxy** — proxy-based credential injection to avoid storing tokens inside containers
- **[Agent version pinning](/reference/roadmap/reproducibility-pinning/)** — pin role repos to tagged versions for reproducible builds, with explicit `--update` to advance (status: open — design proposal)
- **[Auth health and operator visibility](/reference/roadmap/auth-health-and-visibility/)** — pre-launch credential validity probe, `jackin auth status` command, console auth health indicators, and launch-summary diagnostics so auth failures surface before and not during a session (status: open — design proposal)
- **[Reliable Claude authentication strategy](/reference/roadmap/claude-auth-strategy/)** — move beyond copied session auth toward clearer, more durable modes for long-lived and concurrent agent runtimes (status: deferred — needs design work)
- **[Docker runtime hardening contract](/reference/roadmap/docker-runtime-hardening-contract/)** — keep Docker as the default runtime while making its security posture explicit: profiles, DinD hardening, resource budgets, read-only root evaluation, network-policy defaults, and launch-contract reporting (status: open — design proposal)
- **[OrbStack isolated machine backend](/reference/roadmap/orbstack-isolated-machine-backend/)** — macOS backend research for running jackin' role containers and private Docker workflows inside OrbStack isolated machines, with explicit shares and honest shared-kernel risk language (status: open — research and design proposal)
- **[smolvm backend research](/reference/roadmap/smolvm-backend-research/)** — open-source Rust/libkrun VM backend research for true VM-per-workload isolation, OCI-image execution, network-off defaults, allow-host egress, possible future SDK embedding, and a direct comparison with OrbStack isolated machines and Docker Sandboxes (status: open — research and design proposal)
- **[Selectable sandbox backends](/reference/roadmap/selectable-sandbox-backends/)** — umbrella for Docker, OrbStack isolated machines, Kubernetes, and later true microVM providers; detailed backend work now lives in focused roadmap items (status: deferred — umbrella design)
- **[Devcontainer parity](/reference/roadmap/devcontainer-parity/)** — comparison with the upstream `anthropics/claude-code` devcontainer setup; informs network policy + reproducibility decisions (status: proposed — research captured, no implementation committed)

### Reactive daemon program

- **[jackin' daemon — umbrella](/reference/roadmap/jackin-daemon/)** — introduce the long-running per-operator-user host process jackin' will use for reactive features. One umbrella item that decides lifecycle, install method, control socket, security posture, and log redaction once so each reactive feature plugs into one daemon shape. The full list of phase-2/phase-3 reactive adapters lives in the program doc and the sidebar under **Reference → Roadmap → Reactive daemon program** (status: open — design proposal)
- **[jackin' Desktop Agent Hub](/reference/roadmap/jackin-desktop-agent-hub/)** — native macOS menu bar and desktop companion for active jackin' workspaces, isolated agent sessions, PR jump links, and built-in-runtime account state (Claude, Codex, Amp, Kimi, OpenCode), with CodexBar and OpenUsage as scoped references for account quota surfaces. Keeps the agent TUIs as the primary agent UI while using the daemon as the shared state/event backend (status: open — design proposal)
- **[Live bidirectional auth sync](/reference/roadmap/live-auth-sync/)** — Phase 2 adapter. Keep host and every running container in lock-step on each auth axis (`gh`, Claude, Codex, Amp, …). Subsumes the launch-time `sync` mode's bidirectional follow-up; reconsiders the `sync` name in the process (status: open — design proposal)
- **[Agent runtime status authority](/reference/roadmap/agent-runtime-status/)** — Phase 2 state source. Herdr-class in-container authority for `working`, `blocked`, derived `done`, `idle`, `unknown`, and stuck diagnostics, informed by multicode, CCManager, Agent Session Manager, WezTerm Agent Deck, ccmux, TUICommander, Codemux, and tmux-agent-status. Uses semantic runtime hooks/APIs, foreground-process ownership, visible-screen signals, shell markers, and cursor/readiness probes instead of treating PTY silence as attention-needed (status: open — design proposal)
- **[Agent attention prompts](/reference/roadmap/agent-attention-prompts/)** — Phase 2 adapter. Host-side OS notifications when an agent inside a Capsule-managed role container is waiting on operator input or has finished unseen work. Consumes the agent runtime status authority, escalates from silent toast to sound after a configurable timeout, and focuses the right terminal tab where supported. Targets the biggest operator throughput drag: idle wall-clock waiting on agents that don't surface their waiting state (status: open — design proposal)
- **[Host bridge — secrets and approved host actions](/reference/roadmap/host-bridge/)** — Phase 3 adapter. Operator-mediated channel for agents to request a single secret value or invoke a single host command without tearing down the container. Auto-registered MCP server, TouchID / polkit / password approval per request, per-workspace allowlist / blocklist / disabled policies, audit log. Closes the *"agent needs one thing from the host, mid-session"* gap that today forces full-restart workarounds (status: open — design proposal)
- **[Container credential exposure — beyond env injection](/reference/roadmap/container-credential-exposure/)** — Phase 3 follow-on. Auth tokens and operator env values resolved from `op://` references become agent-readable when exported into the container today. Captures the trajectory from documented exposure to file-mount compatibility, command-scoped secret handles, Docker Sandboxes-style credential proxying, and daemon-mediated `secret.request` / `secret.run` grants where the agent gets a handle rather than the raw value (status: open — design proposal)

Default dangerous-mode behavior preserved; role authors can override or extend (status: open — design proposal)
- **[Host affordance bridge](/reference/roadmap/clipboard-image-bridge/)** — host-owned attach client and typed Capsule host-affordance frames so raw host screenshots, copied browser images, explicit image-path staging, GitHub PR/CI URLs, safe visible links, container-file exports, and later file/drop uploads behave like host-local actions across same-kernel Docker, Docker Desktop, and OrbStack while preserving the container boundary and never exposing general host clipboard, browser, or filesystem access to agents; Phase 0 evidence is captured in the [Host Affordance Bridge Preflight](/reference/roadmap/host-affordance-bridge-preflight/) checklist and an opt-in host attach path now exists behind `JACKIN_HOST_ATTACH=1` with shared protocol framing, `attach-proxy`, direct/proxy transport selection, shared terminal-color probing, macOS PNG/Finder plus Linux `wl-paste`/`xclip` clipboard-image staging under `/jackin/run/clipboard/`, chunked large-image transfer, GitHub PR/CI host-open actions, `JACKIN_OPEN_LINKS` opt-out, minimal chunked file export to `~/Downloads/jackin/<instance>/` with explicit reveal/open and under-cursor variants, and an operator-facing [Host Affordances](/guides/host-affordances/) validation guide (status: partially implemented)
- **[Public attribution and project growth](/reference/roadmap/public-attribution-and-growth/)** — research how jackin' can use visible software-development surfaces, especially Git commit trailers and PR metadata, to make the project discoverable without silently mutating operator repositories or misusing authorship semantics (status: open — research and design proposal)

### Codebase health

- **[Workspace-native PR verification](/reference/roadmap/pr-verification/)** — a console flow that builds the per-PR Verify-locally loop into the console: select an instance that's working on a pull request, see its PR details, switch the sidebar into review mode, and run the build/test/serve commands the PR's changes call for — host-side against an isolated checkout, so jackin' can verify its own PRs without touching the operator's real setup or live instance (status: open — design proposal)
- **[Test infrastructure & behavioral specs](/reference/roadmap/test-infra-behavioral-specs/)** — extract shared test helpers into a `jackin-test-support` crate, split mega test files, wire snapshot testing, write behavioral specs for all major components (capsule daemon, operator console, launch pipeline, auth forwarding), and add `proptest` + `cargo-fuzz` for parser surfaces (manifest validation, env resolver, migration chain) (status: open — design proposal)
- **[Visual snapshot testing (CLI & TUI)](/reference/roadmap/visual-snapshot-testing/)** — canonical home for visual / styled-snapshot regression testing of every rendered surface (operator console, capsule, launch TUI, and plain CLI output). Capture the full styled cell (glyph + colours + every modifier), store deterministic SVG goldens that GitHub renders inline so changes show as a before/after diff in the pull request, and keep the whole pipeline Rust-only (extend the first-party `buffer_to_svg` encoder for TUI; evaluate `term-transcript` for CLI output). Absorbs the former *Snapshot tests for TUI render* item; a thin text-only `insta` net and the component SVG lookbook have shipped (status: open — research and design proposal)
- **[CI/CD speed roadmap](/reference/roadmap/ci-speed-roadmap/)** — run-level CI optimization program for faster PR feedback, main-branch CI, preview publishing, release builds, Rust compiler caching, Docker Buildx caching, Bun/docs checks, mise tool caches, nextest scheduling, and dual-runner strategy without dropping coverage; the current branch ships the hosted GitHub default plus optional Velnor parity baseline, while follow-up work tracks remaining warm-runner cache hygiene such as Docker E2E target/toolchain reuse (status: implemented baseline -- follow-up measurement open)
- **[Rust CI tooling and dependency hygiene](/reference/roadmap/rust-ci-tooling/)** — umbrella for Rust CI layers beyond fmt/clippy/nextest/MSRV. Dependency hygiene has shipped with `cargo-audit`, `cargo-deny`, and `cargo-shear`; Codebook spell-check now runs as two whole-branch docs/source jobs through `codebook-lsp lint`; coverage and snapshot expansion remain open. Phase 2 (at/after first release) keeps `dist` for the Homebrew-tap + `cargo-zigbuild` release pipeline and `git-cliff` for changelogs. Includes a verified license audit (zero GPL/AGPL today — the ban is a preventive guard) and records why `release-plz`, `cargo-semver-checks`/`cargo-public-api`, `cargo-vet`/`cargo-crev`, `cargo-mutants`, and `cargo-geiger` were skipped or deferred for a binary CLI workspace (status: partially implemented)
- **[Open review findings catalog](/reference/roadmap/open-review-findings/)** — living backlog of review findings with accepted-exception annotations. Code review and automated scanning consult this catalog (see <RepoFile path="AGENTS.md">AGENTS.md</RepoFile> → "Code review & automated scanning").
- **[Unify the settings and workspace-editor surfaces](/reference/roadmap/unify-settings-editor-surfaces/)** — the global settings screen and the workspace editor edit the same config domains (auth, mounts, env/secrets, trust, general) but are implemented as two near-duplicate surfaces, so a change to one and not the other produces a behavior that exists on one screen and not the other; collapse them into one shared config-surface subsystem (one modal enum, one per-domain edit flow over a scope host trait) so changes propagate by construction; the auth source-folder slice has shipped, the full unification is open (status: partially implemented)

## Contributing

jackin' is open source under the Apache 2.0 license. To develop and test jackin' itself, use [The Architect](https://github.com/jackin-project/jackin-the-architect) — a dedicated agent with the full Rust toolchain:

```bash
jackin load the-architect
```

The project uses `cargo-nextest` for testing and requires all clippy lints to pass before committing.
