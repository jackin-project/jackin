## Guiding principles (binding for every item here)

1. **Correctness over ROI.** Fix what is wrong or inconsistent; never skip an item as "low value". The only reason to stop short is a *proven* impossibility.
2. **Root-cause, architectural fixes.** Every bug means the architecture permitted it. Remove the structural condition and fix the whole class, not the one symptom.
3. **Reuse, never copy-paste.** Recurring concerns (tab focus, hover, cursor, blocking I/O, per-provider headline) are fixed once as a shared mechanism applied everywhere.
4. **Never block.** Blocking work (network, subprocess, DB, file I/O) never runs on the event/input/render task ΓÇõ tokio / `spawn_blocking` / workers, independent loads joined.
5. **CodexBar is a reference to verify, not gospel.** Mirror [CodexBar](https://github.com/steipete/codexbar) by default, but verify every datum against its source; where jackin' has better data, diverge and document why.

- Row 1: provider name (left) ┬╖ account email (right; omit when the provider exposes none).
- Row 2: `Updated <rel> ┬╖ <state>` (left) ┬╖ `<username> ┬╖ <plan>` (right; username only when distinct from email).

## P3 ΓÇõ Usage-state lifecycle: never show a misleading state (F1ΓÇôF3)

The usage segment must follow agent state, identically for **every** agent:

1. **Before the agent has started** ΓÇõ hide the usage segment entirely (today it shows `usage unavailable` during container init, before the agent exists).
2. **Agent started, usage not yet known** ΓÇõ show `refreshing` (one consistent word everywhere), not `usage unavailable`.
3. **Resolved** ΓÇõ the real headline.
4. **Genuinely unavailable** ΓÇõ only then `usage unavailable` (or the specific honest state: `needs login`, `stale`, `unsupported`, `daemon disconnected`).

The segment is **clickable in the unknown/refreshing state** and a click forces a proactive load. Detect agent-start reliably by **routing the agent launch through jackin-capsule as a proxy/hook** (instead of exec-ing the agent directly) so the capsule knows the exact start moment and can trigger auth-verify + refresh. Honest degradation states (never a fabricated number):

| State | Rendering |
|---|---|
| Before agent start | usage segment hidden |
| Refreshing / unknown | `refreshing` (clickable ΓÁÈ force load) |
| Fresh | real quota headline |
| Stale | `stale` + `updated Xm ago`, keep prior data |
| Needs login / secret | the repair action name; no hidden repair |
| Unsupported | `unsupported` + reason |
| Daemon disconnected | `usage unavailable: daemon disconnected` |
| No focused agent | `usage unavailable: no focused agent session` |

## P5 ΓÇõ Shared tab-bar focus model + remaining visual (F14 / F15, Issue 1, Issue 15)

The tab bar reuses the shared `TabStrip` (`crates/jackin-tui/src/components/tab_strip.rs`), the same component as the console settings / workspace-editor surfaces. The model ΓÇõ already specified for that surface ΓÇõ must be applied identically on **every** tab bar in jackin' (console settings, workspace editor, capsule agent-tab bar, capsule usage overlay, status bar, lookbook):

- The **bright phosphor-green** accent is the single focus indicator and always marks where focus is ΓÇõ never in two places at once.
- **Tab bar focused:** the active tab shows the bright-green underline; `Left` / `Right` switch tabs; `Down` / `Tab`, or clicking the content, moves focus into the content.
- **Hover** a tab: the tab color changes AND the cursor changes to a pointer; clicking a tab moves focus onto the tab bar.
- **Content focused:** the tab underline drops to a neutral/dim color (not bright green) and the content block carries the green focus indicator instead.

**Capsule does not match today** ΓÇõ neither the top agent-tab bar nor the usage overlay strip shows the green-on-focus underline, arrow-nav while focused, the hover cursor, or the focus transition into content. Fix both to mirror the console settings reference exactly. This makes `TabStrip` own render + hover + per-tab hit-test + cursor + focus underline, consumed everywhere (Issue 1), and unifies the focus model across surfaces (Issue 15).

**Still owed (capture + verify in a live smoke):** hover color + pointer cursor on every tab surface; status-bar consistency on every screen per the display matrix (debug + non-debug); no vertical jump on tab switch, centered strip, correct symmetric padding (content padded both sides, separators full-bleed); left/right tab switch instant even mid-refresh with no refetch within the TTL.

## Reference ΓÇõ CodexBar extraction recipes (for P0 / P1)

The background refresher resolves a credential, calls the provider usage endpoint, and maps the response to quota buckets; render paths never call these. Only the recipes an open item still needs are kept here.

- **Anthropic / Claude.** Credential `~/.claude/.credentials.json` (`claudeAiOauth.accessToken`), forwarded from the host macOS Keychain service `Claude Code-credentials`. Call `GET https://api.anthropic.com/api/oauth/usage` with `Authorization: Bearer`, `anthropic-beta: oauth-2025-04-20`, `User-Agent: claude-code/<version>`. Windows: `five_hour`ΓÁÈSession, `seven_day`ΓÁÈWeekly, `seven_day_sonnet`ΓÁÈSonnet, `seven_day_opus`ΓÁÈOpus, `seven_day_routines` (aliases `seven_day_cowork` / `claude_routines` / `routines`)ΓÁÈDaily Routines, `extra_usage`ΓÁÈExtra usage; each window carries `utilization` + `resets_at`. Plan prefers `subscriptionType`, falls back to `rateLimitTier`. Account email is NOT in this response ΓÇõ read `~/.claude.json` `oauthAccount` (F12).
- **OpenAI / Codex.** Credential `$CODEX_HOME/auth.json` (default `~/.codex/auth.json`). Call `GET https://chatgpt.com/backend-api/wham/usage` with `Authorization: Bearer` + `ChatGPT-Account-Id`; or launch `codex -s read-only -a untrusted app-server` and call JSON-RPC `account/read` + `account/rateLimits/read`. Map `rate_limit.primary_window`ΓÁÈSession, `secondary_window`ΓÁÈWeekly; `additional_rate_limits[]` `spark`ΓÁÈCodex Spark 5-hour/Weekly; `credits`ΓÁÈCredits; `plan_type`ΓÁÈplan (`Pro 20x`). Identity from the app-server RPC or `id_token` JWT, not a `tokens.email` field.
- **Claude credits (F8).** `claude.ai/api/organizations/{org}/usage` ΓÇõ spent, monthly cap, balance (currency-localized).

### Credential delivery into the container

The capsule runs as PID 1 inside the Linux container and cannot read the host Keychain; the runtime reads the host credential (file or Keychain) and forwards it into the container at a fixed path per provider (`crates/jackin-runtime/src/instance/auth.rs`). File-based providers (Claude, Codex, Grok, Amp) depend on that forwarding; env-key providers (Z.AI, Kimi, MiniMax) are carried in as environment variables regardless of mount mode. P0 depends on the Claude OAuth credential being forwarded reliably so the OAuth path can win over `cli`.
