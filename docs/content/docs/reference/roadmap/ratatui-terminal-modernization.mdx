title: "Ratatui terminal modernization"
---

**Status**: Open — implementation decisions finalized. This item is no longer an upgrade task: jackin' already depends on Ratatui 0.30.2. The work is to make the `jackin-term` model, Ratatui buffer mapping, and Capsule emit path share one explicit terminal contract, then remove the current full-frame repaint stopgap and use Ratatui's modern buffer/diff APIs correctly.

**Implementation scope**: complete this roadmap item in one implementation branch and one large pull request. Do not split phases into separate PRs unless the operator explicitly changes this rule. Use the checklist below as the PR ledger and update completed boxes as work lands.

## Problem

jackin' depends on Ratatui for rich terminal surfaces and on `jackin-term` for the Capsule PTY terminal model. Ratatui's `Buffer` is a compositor buffer, not a PTY emulator: it does not own scrollback, terminal modes, DSR replies, OSC/CSI policy, or the distinction between bytes that update the pane grid and bytes returned to the agent PTY. `jackin-term` remains the owned terminal model.

The current gap is at the seam:

```text
PTY bytes
  -> vte::Parser
  -> jackin_term::DamageGrid
  -> GridView / GridSnapshot / SnapCell
  -> PaneBodyWidget
  -> ratatui::Terminal
  -> SocketBackend
  -> ClientWriter
```

Today that seam has three concrete problems:

- Width is not a written contract. `jackin-term` uses `unicode-width`, while Ratatui 0.30's buffer width logic has additional terminal-compatibility behavior, including the halfwidth-katakana dakuten/handakuten case. A self-consistent but wrong width rule corrupts cursor math, continuation cells, dirty spans, and DSR replies.
- The Capsule used to force a full-frame repaint every frame by filling Ratatui's current buffer with a private-use sentinel before swapping buffers. Phase 3 removes that stopgap so the remaining work can use true diff behavior and meaningful byte-count measurements.
- Terminal-visible metadata is split or dropped: OSC 8 hyperlinks are split between backend regions and PTY passthrough, several SGR attributes are discarded at parse time, and attach-terminal capabilities are not represented as a separate backend concern.

## Baseline

Verified facts this implementation must preserve:

- The workspace is already on Ratatui 0.30.2 through `ratatui = "0.30"`. `termina` and `ratatui-termina` appear only as optional Ratatui dependencies and are not compiled into jackin' binaries today.
- `SocketBackend` already implements the modern Ratatui backend trait shape, including `type Error = io::Error` and all `clear_region` variants.
- Ratatui's flush path already uses a zero-allocation diff iterator. Phase 3 restores the practical byte savings by deleting the sentinel full-frame repaint.
- Existing conformance infrastructure must be extended, not replaced: parser conformance in <RepoFile path="crates/jackin-term/tests/conformance.rs" />, allocation regression in <RepoFile path="crates/jackin-term/tests/allocation.rs" />, fuzzing in <RepoFile path="crates/jackin-term/fuzz/src/damage_grid_process.rs" />, echo-back render conformance in <RepoFile path="crates/jackin-capsule/src/daemon/tests.rs" />, and benches in <RepoFile path="crates/jackin-term/benches/present_frame.rs" />, <RepoFile path="crates/jackin-term/benches/scroll_throughput.rs" />, and <RepoFile path="crates/jackin-capsule/benches/pane_body.rs" />.
- Cursor position, visibility, and DECSCUSR shape are already frame-level state. This work audits and completes that model; it does not rebuild cursor handling from scratch.
- Ratatui 0.30 APIs to use are already available but unused: `CellWidth`, `CellDiffOption`, public `BufferDiff`, and `Terminal::apply_buffer()` / `apply_buffer_with_cursor()`.

## Final Decisions

### 1. Stable virtual-terminal profile, not host-defined semantics

The agent-visible terminal contract is stable for the life of a session. The attached host terminal must not define model width, DSR replies, advertised modes, or color semantics, because an agent can keep running while the operator detaches from one terminal and reattaches from another.

Implement a `jackin-term` virtual-terminal profile as the source of truth for:

- Unicode/display-width table, including dakuten/handakuten, VS16, emoji, ZWJ clusters, combining marks, ambiguous-width code points, and final-column wide behavior.
- The Unicode version implied by that table.
- DECRQM mode-2027 stance.
- DSR cursor replies.
- Agent-visible color model.
- Supported SGR set.
- OSC/passthrough policy.

Implement a separate attach-capability adapter in the backend. It adapts the stable frame to the currently attached terminal or marks the host degraded. Capability information may vary on reattach; it must never flow back into the `jackin-term` model.

### 2. Width work lands before sentinel removal

Width is a correctness bug today; the sentinel only affects diff measurement. Implement the profile-owned width contract first, then retire the sentinel full-frame repaint. Byte-win claims start only after the sentinel is gone.

The width implementation must:

- Add one narrow width module owned by `jackin-term`.
- Derive `Cell::is_wide` and `Cell::is_wide_continuation` from that module.
- Fix `append_to_previous_cluster` so cluster-width mutations update lead width, continuation cells, dirty span, cursor state, pending wrap, and DSR behavior.
- Add an independent hard-coded width oracle in CI. Do not compute the expected table from `jackin-term` or Ratatui.
- Cross-check against Ratatui's `CellWidth` in tests so drift is visible, but keep `jackin-term` free of a runtime Ratatui dependency.

### 3. Ratatui mapping uses `ForcedWidth`, not ordinary continuation `Skip`

Pane rendering must declare model-known width to Ratatui instead of relying on Ratatui to infer width from the symbol.

Mapping rule:

- Lead cells with model width greater than one use `CellDiffOption::ForcedWidth`.
- Ordinary continuation cells stay blank/default and do not use `CellDiffOption::Skip`.
- `Skip` is reserved only for deliberately invisible protocol-only cells after a test proves no stale clear can be hidden.
- `AlwaysUpdate` is reserved for metadata transitions, especially hyperlink boundary changes. It must never be applied broadly.

Add a buffer well-formedness property before diff: every continuation has a matching lead, no visible cell sits inside a lead's effective width span, a final-column wide glyph is handled explicitly, every `ForcedWidth` equals model width, and no stale continuation survives overwrite/delete/erase/resize.

### 4. OSC 8 hyperlinks become model/frame metadata

Use one hyperlink model for PTY OSC 8 links and jackin-owned UI links. Do not keep the current split where PTY links pass through out of band and UI links live only as backend regions.

Implementation shape:

- `DamageGrid` tracks active PTY OSC 8 hyperlink state.
- Printed cells capture the active link metadata.
- Snapshots carry link metadata.
- UI/dialog links use the same frame metadata shape.
- `SocketBackend` emits OSC 8 open/close transitions from frame metadata.
- Link metadata changes over unchanged glyphs force only the affected boundary cells with `AlwaysUpdate`.

### 5. SGR state is parsed completely enough for visible fidelity

Stop dropping terminal-visible SGR state at parse time. Extend `Attrs` and snapshots for underline color, underline style, strikethrough, blink, conceal, and overline, including resets. Preserve SGR subparameters so colon forms such as `4:2`, `38:2:r:g:b`, and `58:2:r:g:b` parse correctly.

Mapping rule:

- Ratatui-native fields map into Ratatui cells directly.
- Attributes Ratatui cannot represent, such as underline style variants and overline, travel as frame metadata consumed by `SocketBackend`.
- Product policy may suppress emission for a visible attribute, such as blink, only if the model still records the state and the docs name the suppression explicitly.

### 6. Terminal state and capabilities get one matrix

Create a documented state matrix that classifies terminal-visible state as one of:

- profile-stable agent-visible state,
- host-adaptive attach capability,
- frame-level reconciled output state,
- boundary-only out-of-band state.

The matrix must cover cursor row/col, cursor visibility, cursor shape, cursor color, cursor blink, mouse mode, focus reporting, bracketed paste, Kitty keyboard / `modifyOtherKeys`, OSC 52 policy, title/CWD OSCs, color depth, OSC 8 support, `?2026`, underline-style support, and image-protocol support.

Reattach may update only host-adaptive rows.

| State | Owner | Reattach behavior |
|---|---|---|
| Display width table, including dakuten/handakuten, VS16, emoji, ZWJ, combining marks, ambiguous width, and final-column wide behavior | profile-stable agent-visible state | Must not change |
| DSR cursor replies | profile-stable agent-visible state | Must not change |
| DECRQM mode-2027 answer | profile-stable agent-visible state | Must not change |
| Agent-visible color model (`TERM=xterm-256color`, `COLORTERM=truecolor`, RGB stored in model) | profile-stable agent-visible state | Must not change |
| OSC 10/11 reported palette values | host-adaptive attach capability feeding model query replies | May update when attach supplies colors; `None` keeps previous known values |
| Supported SGR model state | profile-stable agent-visible state | Must not change |
| Cursor row/col | frame-level reconciled output state | Must not change from attach capability |
| Cursor visibility | frame-level reconciled output state | Must not change from attach capability |
| Cursor shape (`DECSCUSR`) | frame-level reconciled output state | Must not change from attach capability |
| Cursor color (`OSC 12`) | frame-level reconciled output state, pending Phase 5 audit | Must not change from attach capability until modeled |
| Cursor blink | frame-level reconciled output state, pending Phase 5 audit | Must not change from attach capability until modeled |
| Mouse mode | boundary-only out-of-band state | Reassert/reset at frame boundary only |
| Focus reporting | boundary-only out-of-band state | Reassert/reset at frame boundary only |
| Bracketed paste | boundary-only out-of-band state | Reassert/reset at frame boundary only |
| Kitty keyboard / `modifyOtherKeys` | boundary-only out-of-band state | Reassert/reset at frame boundary only |
| OSC 52 clipboard policy | host-adaptive attach capability plus product policy | May vary by host/policy; must not alter PTY grid semantics |
| Title/CWD OSCs (`OSC 0/1/2/7`) | frame-level or boundary-only output state depending on source | May be reasserted at frame boundary; must not alter model width/DSR |
| Color depth | host-adaptive attach capability | May change emitted bytes only after downsampling lands |
| OSC 8 support | host-adaptive attach capability | May change emitted hyperlink bytes only after model metadata lands |
| `?2026` synchronized output support | host-adaptive attach capability | May change frame bracketing/metrics only; unsupported hosts must not alter model |
| Underline-style support | host-adaptive attach capability | May change emitted bytes only after SGR metadata lands |
| Image-protocol support | host-adaptive attach capability | Unsupported for this item except explicit deny/strip policy |

### 7. Scroll optimization records source `ScrollOp`s only

If scroll-region optimization ships in this item, it must be based on source-recorded scroll operations from `DamageGrid`, not inferred from final Ratatui buffers.

Adopt DECSTBM emission only when all gates pass:

- Scroll rect is safe for the physical terminal geometry.
- Capability adapter says the host supports the needed behavior.
- Ratatui's previous/current buffers and physical screen state remain coherent.
- Echo-back conformance passes across pane chrome, dialogs, scrollback view, primary screen, alternate screen, resize, and focus swaps.
- Emitted-byte counters prove a real win on scroll-heavy fixtures.

If any gate fails, leave scroll-region emission out and keep ordinary Ratatui diff output.

### 8. Track soft/hard wrap metadata now; defer full reflow

This item must add row wrap provenance needed for future primary-scrollback reflow. It does not need to implement full reflow.

Track whether each row is a soft wrap of the previous logical line. Preserve that metadata through scrollback, resize bookkeeping, snapshots, and tests. Full primary-screen reflow on resize is a separate follow-up once metadata exists. Alternate-screen reflow remains out of scope.

### 9. Keep Crossterm and defer Termina

Do not migrate to Termina in this item. Crossterm 0.29 remains the raw terminal I/O backend. Termina can be evaluated later as a separate I/O-boundary prototype; it is not part of this implementation plan and never replaces `jackin-term`.

### 10. Inline images are out of scope

Do not implement Kitty, Sixel, or iTerm2 image rendering in this item. Add only the policy needed to avoid unsafe raw passthrough: image protocol bytes are either ignored/stripped or recorded as unsupported media without violating the one-writer invariant. Full modeled media rendering belongs in a separate roadmap item.

### Phase 3 — Remove Sentinel Full-Frame Repaint

- Inventory every byte writer outside Ratatui cell output and frame-state reconciliation.

- Remove the sentinel fill/swap path from `compose_ratatui_frame()`.

- Width oracle and Ratatui cross-checks.
- `append_to_previous_cluster` width mutation, dirty span, cursor, pending-wrap, and DSR behavior.
- Mapping tests from `jackin_term::Cell` / `SnapCell` to Ratatui `Cell` + `CellDiffOption`.
- Parser round-trips for SGR set/reset and colon-subparameter forms.
- State matrix tests for profile-stable vs host-adaptive behavior.

Integration and conformance tests:

- Extend parser conformance with width, SGR, hyperlink, and wrap-provenance fixtures.
- Extend the fuzz corpus with the new cluster classes.
- Extend Capsule echo-back tests with width, hyperlink, SGR, sentinel removal, and scroll-op cases.

- Cover detach/reattach between different terminal programs.

## Explicit Non-Goals

- Do not replace `DamageGrid` with Ratatui `Buffer` or Termina.
- Do not revive direct `GridPatch` ANSI emission.
- Do not adopt Ratatui's `scrolling-regions` feature for the Capsule; it only affects `Terminal::insert_before`, which this path does not use.
- Do not use `CellDiffOption::Skip` for ordinary wide continuations.
- Do not apply `CellDiffOption::AlwaysUpdate` broadly.
- Do not migrate from Crossterm to Termina.
- Do not implement inline image rendering.
- Do not implement full primary-scrollback reflow in this item.
- Do not add a runtime feature flag as the validation strategy.
