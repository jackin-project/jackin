## Prerequisite decisions (operator sign-off)

Three judgment calls gate the rest of Stage 1 / Stage 4. Each ripples across many files, so decide once before any code moves. None should be resolved silently by an agent.

- [x] **Core error-type policy.** Decision: **(a)** add `anyhow` to `jackin-core` and move the traits as-is. Low friction, no consumer-site ripple. The typed-error rewrite would touch bollard impl conversions unnecessarily.
- [x] **`JackinPaths` purity vs ergonomics.** Decision: **(a)** add `directories` + `anyhow` to core, move `detect()` in, keep `ensure_base_dirs()` as a binary-side free fn. Preserves ergonomics across 73 consumers.
- [x] **Selector prefix source.** Decision: **(a)** add `jackin-protocol = { path = "../jackin-protocol" }` to `jackin-core`; re-export `CONTAINER_PREFIX` / `CONTAINER_PREFIX_DASH`. No literal drift; `jackin-protocol` is serde-only and acyclic with core.

## Target repository structure (end state)

The program is finished when the workspace is a **tiered DAG**: a small, stable, dependency-light vocabulary at the bottom; the heavy machinery that reads/writes/orchestrates it in the middle; a thin binary on top. Cargo forbids cycles between crates, so every Stage-4 extraction is converging on this shape. Dependencies point **only downward** — a higher tier may name a lower tier, never the reverse.

```text
Tier 5  jackin (binary)            cli/ · app/ · main.rs — thin CLI dispatch only
        ───────────────────────────────────────────────────────────────────────
Tier 4  jackin-runtime            runtime/ instance/ isolation/ derived_image/
        + TUI surface crates:     + BollardDockerClient + ShellRunner (concrete I/O)
        jackin-console (host TUI) · jackin-launch · jackin-capsule
        ───────────────────────────────────────────────────────────────────────
Tier 3  jackin-env               operator_env / 1Password reference resolution
        ───────────────────────────────────────────────────────────────────────
Tier 2  jackin-config            AppConfig · WorkspaceConfig · ConfigEditor ·
                                  migrations · planner · mount/workspace validation
        ───────────────────────────────────────────────────────────────────────
Tier 1  jackin-tui               shared ratatui widgets + named theme/style constants
        ───────────────────────────────────────────────────────────────────────
Tier 0  jackin-core              Agent · MountIsolation · AuthForwardMode ·
        jackin-protocol          EnvValue/OpRef/FieldTarget · RoleManifest types ·
        (parallel leaves)        Selector · JackinPaths · DockerApi/CommandRunner
                                  traits + data types · constants
                                  ──  NO tokio · NO docker · NO filesystem  ──

Dev/build-only crates outside the tiers: jackin-build-meta, jackin-tui-lookbook.
```

### Per-crate ownership contract

| Crate | Owns (end state) | May depend on | Must **not** depend on | Heavy deps (tokio/docker/fs)? |
|---|---|---|---|---|
| `jackin-core` | universal vocabulary: `Agent`, `MountIsolation`, `AuthForwardMode`, `EnvValue`/`OpRef`/`FieldTarget`, `RoleManifest` + manifest types, `Selector`/`RoleSelector`, `JackinPaths`, `DockerApi`/`CommandRunner` **traits** + their data types, constants | `serde`, `thiserror`, (decision) `directories`/`anyhow`/`jackin-protocol` | any other workspace crate | **No** |
| `jackin-protocol` | capsule↔client wire protocol, `CONTAINER_PREFIX*` | `serde`, `serde_json` | any other workspace crate | No |
| `jackin-tui` | shared ratatui widgets, `theme.rs` style constants, `scroll` primitive | `jackin-core`, `ratatui` | config/env/runtime | No |
| `jackin-config` | `AppConfig`, `WorkspaceConfig`, `MountConfig`, `*Config`, `ConfigEditor`, migration chain + engine, planner, `validate_mounts`, workspace resolution, `resolve_mode*` | `jackin-core`, `toml`/`toml_edit`/`fs2` | env, runtime, instance, isolation | fs only (config read/write) |
| `jackin-env` | `operator_env`: `OpCli`, `OpRunner`/`OpStructRunner`/`OpWriteRunner`, `EnvLayer`, `resolve_operator_env`, `validate_reserved_names`, launch diagnostics | `jackin-core`, `jackin-config`, `serde_json` | **`jackin-console`** (see cleanup below), runtime, instance | subprocess (`op` CLI) |
| `jackin-runtime` | `runtime/`, `instance/`, `isolation/`, `derived_image/`, `detect_workspace_edit_drift`, concrete `BollardDockerClient` + `ShellRunner` | `jackin-core`, `jackin-config`, `jackin-env` | the binary crate | **Yes** — the only place tokio + bollard + privileged docker live |
| `jackin-console` / `jackin-launch` / `jackin-capsule` | host / launch / in-container TUI surfaces under one Elm Architecture boundary | `jackin-core`, `jackin-tui`, `jackin-protocol`, (console) config/runtime | — | TUI + (console) runtime |
| `jackin` (binary) | `cli/` arg structs, `app/` dispatch, `main.rs`, `bin/role.rs` | all of the above | — | thin glue only |

### DAG invariants — final verification checklist
- [x] The full inter-crate graph is acyclic — `cargo build --workspace` succeeds with no `cyclic package dependency` error; spot-check with a dependency-graph tool
- [x] `jackin-core` carries no `tokio`, no `bollard`/docker, no `*Config` type — `cargo tree -p jackin-core` shows only the sanctioned leaf deps (`ensure_base_dirs` uses `std::fs::create_dir_all` — pure std, no tokio file I/O)
- [x] `jackin-config` does not name `jackin-env`, `jackin-runtime`, `instance`, or `isolation` — `cargo check -p jackin-config` passes standalone
- [x] `jackin-env` does not name `jackin-runtime` — `cargo check -p jackin-env` passes standalone
- [ ] `jackin-runtime` is the **only** crate depending on `bollard` and the privileged-docker code paths
- [ ] The root binary contains only `cli/`, `app/`, `main.rs`, `bin/` glue — every domain `use` resolves to `jackin_{core,config,env,runtime}`
- [ ] **Layering cleanup — break the `jackin-env → jackin-console` edge:** `OpCli` imports the 1Password data aliases (`OpAccount`, `OpVault`, `OpItem`, `OpField`, `OpCache`) from `jackin_console::tui::components::op_picker`. A non-visual logic crate depending on a TUI crate inverts the tier order. Move those plain data types **down** to `jackin-core` (or `jackin-config`) and have both `jackin-env` and `jackin-console` import them from there, so `jackin-env`'s dependencies are exactly `core` + `config`
- [x] Every new crate uses the self-named Rust 2024 layout (`foo.rs` + `foo/`, no `mod.rs`) and enforces `clippy::mod_module_files = "deny"` — verified: 0 `mod.rs` files in all crates, all 4 new crates have the lint

## Rust best-practices acceptance gates (every PR under this program)

These are the per-diff checks a reviewer verifies, drawn from the roadmap's "Conventions every refactor follows" and the repo rules in <RepoFile path="AGENTS.md" />. They are written as pass/fail gates, not aspirations — a PR that fails one is incomplete. They compose with, and do not replace, the structural tasks above.

### Crate boundaries (Stages 1 & 4)
- [x] Public API is deliberate: `pub(crate)` by default, `pub` only on what the next crate up imports — enforced via unreachable_pub pass
- [x] Struct fields stay private unless the representation is part of a stable contract (the serde `*Config` structs expose fields intentionally; most other types do not)
- [x] `jackin-core` stays types + traits only — no `tokio`, no subprocess; `ensure_base_dirs` uses only `std::fs::create_dir_all` (no tokio file I/O, no bollard)
- [ ] Invariants encoded in types — newtypes and enums over `bool` / `String` / bare primitives, so each higher crate inherits the constraint instead of re-checking it
- [x] Standard traits derived where they make semantic sense on the `jackin-core` vocabulary — `Debug`, `Clone`, `PartialEq`/`Eq`, `Hash`, `Default` for data; `Display` + `std::error::Error` for errors; `From` / `TryFrom` for conversions
- [ ] `derive(Serialize, Deserialize)`, public re-exports, and public dependencies treated as intentional compatibility commitments — present only where the serialized shape or the re-export is part of the contract
- [ ] Sealed trait used where a downstream `impl` would block future evolution (the `AgentRuntime` trait — private `sealed::Sealed` supertrait)

### Shared extractions (Stage 2)
- [x] Extracted in a shape future callers reuse — followed throughout this branch (e.g., `provision_single_file_credential` with params, `provision_kimi_dir_credential` with `sync_files: &[&str]`)
- [x] Borrowed parameters when ownership is not required: `&str` over `String`, `&[T]` over `Vec<T>`, `&Path` over `PathBuf`, `Option<&T>` over `&Option<T>`
- [x] `.clone()` is a deliberate design decision with a reason, never a borrow-checker appeasement; no needless intermediate `Vec` / `String` allocations
- [x] Any new third-party dependency is justified in the PR (internal crate splits are free; an external crate carries compile, supply-chain, and API cost) — no new external deps added in this branch

### File splits & functions (Stage 3)
- [x] First-time-reader order: entry-point items first, important types before helpers, implementation near its use — followed in all new module files created this session
- [x] Control flow stays local and visible; preconditions expressed in types or adjacent checks, not a distant helper; clear branching over a clever combinator chain
- [x] Boring, searchable names over abbreviations — followed throughout (e.g., `load_role_manifest`, `validate_role_manifest`, `provision_kimi_dir_credential`)
- [x] Every newly split module gets a `//!` contract — all new files created in this branch have `//!` doc headers

### Errors
- [x] `Result<T, E>` for expected failure; `Option<T>` only when absence carries no useful detail
- [x] **Typed errors (`enum` + `thiserror`) at crate boundaries** — `jackin-core` uses `thiserror` for `ParseAgentError`, `ParseMountIsolationError`, `SelectorError`; `anyhow` only at binary edge (per decision (a))
- [x] `?` for propagation, with context added at the I/O, parse, network, and user-facing boundaries
- [x] `unwrap` / `expect` reserved for tests, unreachable invariants, and programmer errors — no new `unwrap` calls added in this branch
- [x] Public library functions document failure with a rustdoc `# Errors` section where relevant (e.g., `ensure_base_dirs`, `load_role_manifest`)

### Tests & docs (Stages 0 & 3)
