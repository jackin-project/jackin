Internal documentation for contributors and AI agents working on jackin❯ itself. This section covers architecture, design decisions, and behavioral specifications for the subsystems that make up the codebase.

For the high-level codebase map and module layout, see the [Codebase Map](/reference/getting-oriented/codebase-map/). For Architecture Decision Records, see [ADRs](/reference/adrs/). For roadmap and planned work, see [Roadmap](/roadmap/).

## Contents

- [Code Tour](/reference/developer-reference/code-tour/) — key call chains: `jackin load`, `jackin console`, and `jackin hardline`.
- [Behavioral Specs](/reference/developer-reference/specs/) — INV-format invariant contracts for the subsystems targeted for refactor.
