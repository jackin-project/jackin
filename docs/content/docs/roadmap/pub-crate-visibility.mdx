**Status**: Partially implemented — lint enabled, low-hanging-fruit items fixed (2026-06-02); ~100 warnings remain for the second pass.

## Problem

`jackin` has a real library surface: <RepoFile path="crates/jackin/src/lib.rs">crates/jackin/src/lib.rs</RepoFile> publicly exports most top-level modules and <RepoFile path="crates/jackin/src/bin/role.rs">crates/jackin/src/bin/role.rs</RepoFile> consumes the library. That means a blanket "binary crate ⇒ convert everything to `pub(crate)`" pass would be wrong. The actual readability problem is narrower: some items are still wider than needed.

## What shipped (2026-06-02)

- **Lint available**: `unreachable_pub` is intentionally *not* in `[lints.rust]` in <RepoFile path="Cargo.toml">Cargo.toml</RepoFile> — it has ~100 remaining violations and `cargo clippy -- -D warnings` would promote them all to errors. Run `RUSTFLAGS='-W unreachable-pub' cargo check` for the advisory pass. When the remaining items are fixed, the lint can be added to `[lints.rust]`.
- **Low-hanging-fruit items fixed** (all 8 roadmap-specified items, with corrected names):
  - <RepoFile path="crates/jackin-config/src/planner.rs">crates/jackin-config/src/planner.rs</RepoFile> — `WorkspaceCreatePlan`, `WorkspaceEditPlan`, `plan_create`, `plan_edit`, `apply_isolation_overrides` → `pub(crate)`
  - <RepoFile path="crates/jackin-config/src/app_config/mounts.rs">crates/jackin-config/src/app_config/mounts.rs</RepoFile> — `MountEntry` → `pub(crate)`
  - <RepoFile path="crates/jackin-config/src/app_config/roles.rs">crates/jackin-config/src/app_config/roles.rs</RepoFile> — `BUILTIN_ROLES` (name changed from `BUILTIN_AGENTS`) → `pub(super)`
  - <RepoFile path="crates/jackin-runtime/src/runtime/launch.rs">crates/jackin-runtime/src/runtime/launch.rs</RepoFile> — `inspect_attach_outcome`, `EnvLayerState`, `LaunchError`, `verify_credential_env_present` → `pub(super)`
  - <RepoFile path="crates/jackin-runtime/src/runtime/attach.rs">crates/jackin-runtime/src/runtime/attach.rs</RepoFile> — `JACKIN_STATUS_CMD`, `parse_session_count`, `docker_unavailable_msg` → `pub(crate)` (isolation/finalize.rs and app/mod.rs access these from outside runtime; pub(super) would break those paths)

## Remaining work

~100 warnings remain across 26 files (run `cargo check 2>&1 | grep "unreachable_pub"` for the current set). The largest clusters are in private `console/` sub-modules (`domain`, `services/`) and `runtime/universe.rs`. These require deciding the intended visibility per-item before bulk-converting.

The next pass should:
1. Decide the intended library surface in <RepoFile path="crates/jackin/src/lib.rs">crates/jackin/src/lib.rs</RepoFile> — what truly needs to stay `pub` for `main.rs`, `jackin-role`, and tests.
2. Work through the private-module clusters (console services, runtime universe) where `pub(super)` is the obvious narrowing.
3. Verify `cargo nextest run` still passes after each batch.

## What stays `pub`

- `pub fn run` and other intentionally exported library entrypoints used by sibling binaries
- validation helpers consumed by <RepoFile path="crates/jackin/src/bin/role.rs">crates/jackin/src/bin/role.rs</RepoFile>
- any types that are deliberately part of the internal library boundary exposed from <RepoFile path="crates/jackin/src/lib.rs">crates/jackin/src/lib.rs</RepoFile>

## Caveats

Do not treat `unreachable_pub` as a turnkey "convert hundreds of bare `pub` items" script. It is a cleanup aid for the current exported shape, not a substitute for deciding what the library boundary should be.
