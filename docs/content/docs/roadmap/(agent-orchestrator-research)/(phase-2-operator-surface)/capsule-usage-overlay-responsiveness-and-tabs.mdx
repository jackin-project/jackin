**Status**: Partially implemented — provider quota/account surfaces, status-bar headline mapping, provider verification, cache-backed refresh, honest stale/login/unsupported states, real reset epochs, and the focused-agent tab model exist; remaining work is provider-fidelity cleanup, the shared `TabStrip` fold, and live visual acceptance smoke.

## Current shipped surface

`jackin-capsule` has a read-only usage overlay with Overview plus provider tabs for Claude, OpenAI/Codex, Amp, xAI/Grok, Z.AI, Kimi, and MiniMax. The shipped surface includes provider account identity, `Auth:` origin rows, reset windows, status-bar headline labels, cache-backed refresh, provider verification, and real `resets_at` reporting on the CLI path. Reduced provider data is no longer labelled authoritative.

Durable operator documentation belongs in the Capsule reference. This roadmap page tracks only the remaining implementation work.

## Remaining work

1. **Focused-agent start/refresh lifecycle.** Confirm the usage segment stays hidden before the focused agent exists, renders a deliberate refreshing state while data is loading, and does not flash `usage unavailable` during normal startup.
2. **Provider drift from `usage verify`.** Treat any provider response/schema drift found by `jackin-capsule usage verify` as correctness bugs. Fix the resolver or display model, not the one rendered symptom.
3. **Shared `TabStrip` fold.** Replace the hand-rolled provider-tab painter with the shared `TabStrip` once the shared component has the glyph/support needed for the usage dialog. This is a visual-risk refactor, so keep it separate from provider-data fixes.
4. **Live visual acceptance smoke.** Run the interactive `--debug` visual pass for the focused-agent model and provider tabs. The JSON output is not authoritative for visual acceptance; the TUI render is.

## Invariants

- Never present a reduced snapshot as `authoritative`.
- Omit absent provider data instead of fabricating `100% left` or placeholder identities.
- Keep provider-specific source selection and status-bar labels semantic, not substring-matched.
- Blocking work stays off the input/render task.
- Reuse shared TUI components when they can express the same visual contract.
