**Status**: Active. This page is the implementation plan and acceptance contract for routine codebase-health work.

## Goal

Make jackin❯ easy to refactor by humans and AI agents: compiler and tooling gates catch predictable bugs before review, crate and module boundaries make the right edit location obvious, critical behavior has executable coverage, performance hot paths are measured instead of guessed, and every surviving exception is justified.

## Rust enforcement and exception debt

1. **Convert stable broad `#[allow]` suppressions to narrow `#[expect(..., reason = "…")]`.** Retain `allow` only for demonstrated cfg, test, generated-code, or target-specific instability. Set `allow_attributes = "deny"` beside `allow_attributes_without_reason`; if compiler or cfg behavior makes a specific site unstable, record that exact exception and its removal trigger. Keep the bespoke reason gate only for carve-outs the compiler lints cannot express. Acceptance: each exception has a narrow reason, every incompatibility is enumerated with a removal trigger, and the suppression ratchet has no unexplained growth.
2. **Finish the measured lint ratchets without reducing the census to a hand-picked subset.** Inventory every workspace `allow`; for each lint record the deny/warn/allow decision, measured count and date, rationale, and revisit trigger. The census must include `needless_pass_by_value`, `large_futures`, `assigning_clones`, `match_same_arms`, `drop_non_drop`, `format_push_string`, `option_option`, `struct_field_names`, and every other current or newly introduced allow. Preserve the `unused_self` and `unused_async` warning baseline with the same evidence. Tighten complexity thresholds only after a fresh measured maximum supports it. The target is a justified policy, not a mechanically low count.
3. **Complete restriction-lint adoption decisions.** Roll the slice/index panic family through every crate where its production signal is actionable; decide `map_err_ignore` and the optional lossy-cast tier from a fresh census. Keep `assertions_on_result_states` denied, verify its test escape valves, and remove needless workaround patterns. Keep ergonomic test escape valves aligned with every adopted lint. Pilot the whole-category restriction allowlist posture on one pure crate before considering wider adoption.
4. **Enforce blocked surfaces.** Maintain a reasoned `clippy.toml` inventory of disallowed methods, types, and macros with the canonical replacement on every entry. Expand blocking process/filesystem methods only from a measured render/runtime-path census and keep narrow startup/build/test exceptions. Evaluate `std::sync::{Mutex, RwLock}` on async paths, `Rc<RefCell<_>>` in async code, interior mutability on `Send` boundaries, non-canonical ring buffers, and bare primitive protocol payloads rather than banning them indiscriminately. Production printing, `dbg!`, `todo!`, `unimplemented!`, and prohibited telemetry/process macros must be compiler- or Dylint-enforced where built-in lints are insufficient.
5. **Complete routine custom project enforcement only where a built-in lint cannot express the invariant.** The candidates owned here are foundational API sealing/Debug discipline and schema-field consistency. Run each Dylint candidate advisory first, record its false-positive rate, and promote only useful rules to a pinned CI lane. Direct protected-process bypass and telemetry/export-safety enforcement belong to [Sensitive boundary code health](/roadmap/sensitive-boundary-code-health/); do not create competing routine gates here.
6. **Make suppression measurement trustworthy before using it as a ratchet.** Parse Rust attributes with a syntax-aware parser or robust lexer; add comment, string, raw-string, multiline, and nested-syntax negative cases; generate the baseline and ratchet from real attributes; and prove that `cargo xtask lint suppressions` reports the enforced reason policy.
7. **Make advisory CI evidence complete and actionable.** Decide whether `cargo deny check advisories` runs at PR time. Run and report Miri independently for core, config, manifest, and term. Perform an exit-status-checked hakari analysis with a before/after timing comparison and record an explicit adopt/no-adopt decision. Advisory jobs may remain non-blocking, but every promised result must be visible and distinguish tool failure from a clean result.
8. **Preserve dependency-boundary decisions as executable policy.** Keep `jackin-usage` as the sole Turso SDK import owner; audit direct imports, the Cargo pin rationale and revisit triggers, the cargo-deny license exceptions, and advisory/audit coverage. A dependency pin or exception is not closed merely because it has a comment.
9. **Enforce the container-path rule.** Implement a token-aware absolute-container-path audit for forbidden `/run`, `/var`, `/opt`, `/etc`, and `/tmp/jackin*` paths; distinguish host-only and test fixtures from container emitters; and require narrow reasoned exceptions. Acceptance: each prohibited-root fixture fails with file/line and a fix, every production container path routes through the owned chokepoint or an audited external-system exception, and the allowlist is shrink-only.
10. **Remove the runtime image file-size exception by ownership extraction.** Split only along named image-resolution/build decisions with characterization coverage, then remove the `ratchet.toml` exception. Do not satisfy the budget with arbitrary line movement.

Preserve the `jackin-env` pilot—private implementation modules, curated root exports, and an intentional test-support exception—with a check that prevents downstream module imports from becoming an accidental API. Inventory the module tree and add a structural check only after it can distinguish meaningful contracts from boilerplate.
7. **Decompose the root CLI integration by observable responsibility.** Split the large `load`, Claude-token, and workspace command handlers around setup, dispatch, and effect boundaries without changing CLI behavior or console lifecycle. Add explicit characterization for the bare-command TTY-capability fallback and `launch` deprecation warning in `app.rs`. Remove each `too_many_lines` expectation only after its narrow behavioral coverage remains green.
8. **Keep public-surface growth measurable.** If a checked-in public-API snapshot remains inappropriate for a pre-release CLI workspace, replace it with an executable public-item/root-re-export growth report and a reviewed baseline. A prose decision to skip snapshots is not an alternative gate.

5. **Add deterministic control-plane testing.** Put capsule control/session logic behind injectable ports at the real daemon/session-supervisor boundary, with fake-port tests proving observable behavior rather than only helper predicates; then evaluate turmoil or madsim for replayable scheduling, `proptest-state-machine` for registry and reattach policies, and test-only failpoints at Docker, PTY, filesystem, and process edges. ### OTLP LogRecord wire contract

This is the required contract for every first-party operational log sent through OTLP. Exporter-backed evidence must prove the emitted shape. It follows the OpenTelemetry [Logs Data Model](https://opentelemetry.io/docs/specs/otel/logs/data-model/) and [event conventions](https://opentelemetry.io/docs/specs/semconv/general/events/): a LogRecord has top-level event and observed timestamps, severity, trace context and flags, body, resource, instrumentation scope, attributes, dropped-attribute count, and a stable event name. A named event identifies one stable body-and-attribute structure.

| LogRecord part | Required shape | Prohibited shape |
| --- | --- | --- |
| `Timestamp` / `ObservedTimestamp` | `Timestamp` is when the application event occurred; `ObservedTimestamp` is when the SDK observed it. Preserve source event time when bridging capsule/file records and never silently substitute ingestion time for known event time. | Encoding timestamps in the body, dropping source time during bridging, or using one ambiguous timestamp for both meanings. |
| Resource | Stable source identity only: `service.name`, build/version identity, runtime/platform, and telemetry SDK identity. Resource values must not vary per run, container, session, request, model, or operator. Two runs of the same build must share resource identity while differing in record attributes. | Putting `parallax.run.id`, `session.id`, `jackin.component`, a container id, a model/provider name, command arguments, or a host path in the resource. |
| Instrumentation scope | The emitting crate/instrumentation library and version. Use the tracing target for crate/module filtering; do not invent a scope from the rendered message. | Copying `[jackin debug docker]`, a category, or a model name into scope identity. |
| `EventName` | A non-empty registered, dotted, low-cardinality top-level field such as `docker.container.inspect`, `capsule.attach.failed`, or `launch.plan.selected`. It is the event type and uniquely determines the permitted attribute schema and body intent. If the current tracing bridge cannot populate it, adapt or replace the bridge; a temporary `event.name` attribute must be converted and equality-tested. | Leaving the top-level field empty, deriving it with `kind.replace(...)`, or including a model, provider, container, run id, path, command, URL, error text, or stage interpolation. |
| Body | One concise, redacted, human-readable phrase such as `container inspected` or `capsule attach failed`. The body may explain the occurrence, but never carries the schema. | Bracket prefixes, severity/category labels, timestamps, `key=value` serialization, raw JSON, full argv/URLs, terminal bytes, Docker inspect output, generated files, or a model/provider name. In particular, OTLP must never receive `[jackin …]` or `[jackin-capsule …]` bodies. |
| Severity | Map the typed operation level to both `SeverityText` and `SeverityNumber`; normal milestones are `INFO`, diagnostics are `DEBUG`/`TRACE`, recovered danger is `WARN`, failures are `ERROR`, and fatal process termination is `FATAL`. | Encoding severity in the body or treating an expected detach as an `ERROR` merely because it carries an error value. |
| Trace context and `TraceFlags` | Preserve active `TraceId`, `SpanId`, and W3C `TraceFlags` on every log inside work. A log with a span id must have its trace id. Independent work is connected with a standard span link, not a fabricated child-trace field. | Replacing trace id with run id, exporting logs outside their active launch/session span, or making a backend parse correlation from text. |
| Attributes | Preserve native boolean, integer, floating-point, string, and bounded-array types; validate keys and values through the event registry before export. Dynamic evidence belongs here only when allowlisted and redacted. | Stringifying every value, accepting arbitrary call-site keys, storing schema in `body`, or exporting unbounded collections/payloads. |
| Dropped attributes | Configure explicit SDK limits, expose `DroppedAttributesCount`, and treat unexpected dropping in the conformance scenario as failure. Decide deliberately whether the application or Collector performs each redaction/normalization; the Collector is defense in depth, not permission to emit secrets. | Silent truncation, relying on a backend's undocumented limit, or dropping required event fields while still counting the record as conformant. |

The following attributes are the canonical application schema. `jackin.category` is the one wire key; do not emit `log.category`. The registry owns every key and every allowed event name, so call sites never inline a spelling.

| Attribute | Presence | Value rules |
| --- | --- | --- |
| `event.name` | Bridge compatibility only | Must equal the registered top-level `EventName`; the adapter consumes or mirrors it only until the Rust bridge writes `EventName` directly. It must never become an independent second name. |
| `event.outcome` | Every completed operation/event | Registered outcome: `success`, `failure`, `timeout`, `cancelled`, `cache_hit`, or `expected_close`; no ad-hoc prose. |
| `jackin.component` | Every application event | `host`, `capsule`, or `console`; it is an event dimension, not a body prefix. |
| `jackin.operation` | Operation and lifecycle events | Registered logical operation such as `docker.inspect` or `capsule.attach`; no command text. |
| `jackin.stage` | When a workflow stage applies | Registered stage such as `preflight`, `image`, `run`, `attach`, or `cleanup`. |
| `jackin.category` | Every application event | Registered diagnostic category; body-derived categories and `log.category` are prohibited. |
| `parallax.run.id` | Every application log and span in a run | Wrapper-supplied run id wins over a generated fallback. It is a record/span correlation dimension, never a resource attribute, error fingerprint input, or default metric dimension. Metrics correlate through exemplars/trace context where supported. |
| `session.id` | Capsule/session events | Opaque session correlation value; never part of event name, body, or fingerprint. |
| `error.type` | Failures only | Stable low-cardinality machine class shared by the error log, an attribute on the active span, and the failure metric. Separately set the span status code to `ERROR`; status description is not the error-type field. Do not emit `error_type`; successful events omit the key. |
| Domain evidence | Only when it helps diagnose the event | Allowlisted, redacted, bounded attributes such as `process.command`, `process.exit_code`, `container.name`, or `container.state`. Dynamic values are evidence only, never grouping/fingerprint/name/body inputs. Large or sensitive evidence becomes an `artifact.id`/redacted path reference, not a log attribute. |

Every registered event definition must contain: top-level name; default severity; one stable body intent; required and optional attributes with native types; allowed outcomes; privacy class; cardinality class; sink eligibility; error-fingerprint inputs; and the owning crate. Registration fails closed on unknown keys, missing required keys, prohibited spellings, or a body that violates its redaction/size policy. Schema changes need compatibility review and exporter-backed fixtures; call sites never construct the registry dynamically.

Generic operational telemetry must not include Rust struct/enum/model type names, agent identity, provider identity, or LLM model identity in a resource, instrumentation scope, event name, body, default attributes, or error fingerprint. If a future product feature needs agent or GenAI telemetry, it must be designed separately against the then-current OpenTelemetry GenAI semantic conventions, with explicit consent, privacy, cardinality, retention, and operator-contract review. That future schema is not a compatibility reason to retain model/provider fields in generic operational logs.

A conforming exported record is shaped like this (illustrative JSON; OTLP keeps typed values rather than this display encoding):

```json
{
  "Timestamp": "<event time>",
  "ObservedTimestamp": "<SDK observation time>",
  "Resource": {
    "service.name": "jackin",
    "service.version": "<build version>"
  },
  "InstrumentationScope": {
    "name": "jackin-diagnostics",
    "version": "<crate version>"
  },
  "SeverityText": "ERROR",
  "SeverityNumber": 17,
  "EventName": "capsule.attach.failed",
  "Body": "capsule attach failed",
  "TraceId": "<active trace id>",
  "SpanId": "<active span id>",
  "TraceFlags": "<active W3C trace flags>",
  "DroppedAttributesCount": 0,
  "Attributes": {
    "event.outcome": "failure",
    "jackin.component": "host",
    "jackin.operation": "capsule.attach",
    "jackin.stage": "attach",
    "jackin.category": "capsule",
    "parallax.run.id": "<run id>",
    "session.id": "<session id>",
    "error.type": "attach_socket_eof"
  }
}
```

One typed event builder is the only path to sinks: validate against the registry, redact and bound attributes, attach current trace context, emit the OTLP LogRecord, then render any console/JSONL/capsule-file representation. Console and local-file renderers may add bracket prefixes after this point; exporters must not receive those rendered strings. Migrate existing JSONL deliberately through a versioned adapter: map `kind`, `stage`, `detail`, `error_type`, and `log.category` to the canonical schema, then test that OTLP and canonical-schema JSONL records contain none of those prohibited keys.

### Span, metric, and file-sink contracts

- **Operation spans:** one stable registered name per long or failure-prone work unit; component, operation, stage, bounded evidence, and declared completion fields; duration from span lifetime; registered outcome and `error.type` recorded at completion; OpenTelemetry status `ERROR` on real failure. Dynamic names and failure-only body text are forbidden. Async child work inherits context; independently navigable BuildKit, provider-probe, or capsule-session work starts a separate trace root only when the boundary registry requires it and carries a standard link to the causing span.
- **Metrics:** terminal bytes, render duration/frame/cell counts, interaction counts, usage refreshes, Docker inspect counts, database statement counts, and typed failures are counters or histograms with semantic units and low-cardinality labels—not per-event DEBUG logs. A metric registry pins name, instrument kind, unit, aggregation, and allowed dimensions. Run/session/container/model identifiers are never metric dimensions unless a separately approved bounded contract proves their value.
- **File/JSONL:** the canonical event is structured before formatting. JSONL retains canonical keys and native values plus real trace/span ids where available; the capsule text file receives a context banner and a human renderer may add prefixes. OTLP mode does not fabricate a local artifact path. Backend query hints use the effective run id and configured backend.

Completion requires an exporter-backed matrix, run in CI against a real host-to-capsule path rather than direct facade calls:

1. INFO, DEBUG, WARN, ERROR, FATAL, and expected-close records have the required timestamps, top-level event name/severity/body/context/flags, canonical typed attributes, and zero unexpected dropped attributes; optional attributes are absent when inapplicable.
2. Two runs of one build in the same runtime environment share the stable Resource and instrumentation-scope fields while run/session/component differ on records; across environments, only legitimate runtime/platform fields may differ. Every capture asserts that Resource excludes run/session/component identity. No generic record contains a Rust model/type, agent, provider, or LLM model field.
3. A Docker/HTTP/launch/capsule scenario proves facade attributes survive export and neither sink exposes preformatted `[jackin-capsule …]` text, prohibited keys, or prefixes.
4. Routine OTLP-shape assertions prove model/provider labels, argv, URLs with queries, inspect JSON, terminal bytes, and generated files are absent from generic record bodies and indexed attributes, with usable evidence/artifact references where the registry permits them. Canary-secret corpora, protected-value classification, redaction across every sink/fingerprint/bundle, and their negative-assertion helpers are acceptance dependencies owned by [Sensitive boundary code health](/roadmap/sensitive-boundary-code-health/); this matrix must invoke that conformance gate rather than duplicate its policy.
5. A forced attach failure produces exactly one `ERROR` record with stable `error.type`, matching active-span status and failure metric; run/container/path/user/command variation does not split its fingerprint. Expected detach uses the one registered `expected_close` outcome without an error type; `expected_shutdown` is adapter-only and cannot appear in canonical output.
6. The capture asserts run id, session id, `jackin.screen.name`, inherited trace context, and required span links across host, Docker/process, launch, attach, cleanup, and capsule boundaries. It separately verifies the file-only/JSONL correlation and UI artifact/query contract.
7. Measured default-mode record/span/metric volume—not source constants—is the ratchet input, and high-frequency terminal/render/input activity changes metrics without recreating per-event log firehose.
