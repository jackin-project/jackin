**Status**: Superseded by current design — root contributor files remain canonical entrypoints; Fumadocs links to durable contributor references where they exist

## Current shape

<RepoFile path="CONTRIBUTING.md">CONTRIBUTING.md</RepoFile> and <RepoFile path="TESTING.md">TESTING.md</RepoFile> intentionally remain at the repository root because GitHub, maintainers, and AI agents discover them there. They should not be moved wholesale into Fumadocs.

The docs site now carries durable contributor references under `/reference/`, including [Architecture](/reference/getting-oriented/architecture/) and [Codebase Map](/reference/getting-oriented/codebase-map/). Those pages summarize stable concepts instead of duplicating every root-process rule.

## Remaining work

1. Keep root files as the source of truth for contribution workflow and test commands.
2. Add Fumadocs contributor pages only for durable explanations that benefit from site navigation.
3. Link from root files to canonical docs pages when the docs page expands the topic, but avoid duplicate copies that drift.
4. If a root rule changes, update any summary reference page in the same PR.

## Non-goal

Do not replace <RepoFile path="CONTRIBUTING.md">CONTRIBUTING.md</RepoFile> or <RepoFile path="TESTING.md">TESTING.md</RepoFile> with wrappers. The root files are intentional operational entrypoints.
