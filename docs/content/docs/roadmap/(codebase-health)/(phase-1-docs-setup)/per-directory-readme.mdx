**Status**: Open — update orientation files to match the workspace crate split and Rust 2024 self-named module layout

## Problem

AI coding agents load nearby `README.md` and `AGENTS.md` files for orientation. The Cargo workspace split changed the useful orientation unit from the old root `src/` tree to crate directories under `crates/`, and the workspace now forbids `mod.rs` in crates.

This item remains valid, but the target content needs to describe the current crate layout rather than pre-split paths.

## Target directories

| Directory | README or AGENTS.md should say |
|---|---|
| `crates/` | Workspace-wide Rust rules: self-named modules, tests in `tests.rs`, lint baseline, and dependency hygiene. |
| `crates/jackin/` | CLI binary crate: command dispatch, app context, command handlers, and root integration tests. |
| `crates/jackin-runtime/` | Container bootstrap pipeline, instance lifecycle, launch pipeline, DinD sidecar management, isolation, and runtime tests. |
| `crates/jackin-console/` | Operator console TUI state, input/update, screens, services, and shared console components. |
| `crates/jackin-config/` | Config loading, persistence, editing, schema versions, and migrations. |
| `crates/jackin-env/` | Operator environment resolution, 1Password references, picker metadata, and launch diagnostics. |
| `crates/jackin-capsule/` | In-container capsule PID 1, PTY/session control, attach protocol, and terminal model integration. |
| `docs/` | Fumadocs docs site. Public docs live in `content/docs/`; route groups in parentheses are organizational only. |

## Rules for crate guidance

- Keep each orientation file short: entrypoints and invariants, not a second codebase map.
- Do not create or recommend `mod.rs`; crates use Rust 2024 self-named module files such as `foo.rs` plus `foo/tests.rs`.
- Point durable source navigation to [Codebase Map](/reference/getting-oriented/codebase-map/) and root <RepoFile path="PROJECT_STRUCTURE.md">PROJECT_STRUCTURE.md</RepoFile>.

## Remaining work

1. Audit existing crate-level orientation files for stale root-`src/` language.
2. Add missing crate-level `README.md` or `AGENTS.md` only where they prevent repeated mis-edits.
3. Keep the docs-site and repo-root source maps aligned when a crate boundary changes.
