## Problem

At original analysis, many source files lacked `//!` orientation docs. That module-level gap is now closed for production crate source; this page remains as the shipped design record.

## What shipped (2026-06-02)

Priority production files received `//!` blocks following the `env_model.rs` exemplar format (purpose + scope claim + key invariants). The old per-file checklist is intentionally not repeated here because the crate split and self-named module layout now make stale path tables more expensive than useful.

Future audits should re-run coverage against the current tree instead of copying historical paths.

## Current coverage (2026-06-02)

Production crate source is documented. Excluded intentionally: binary entry points (`main.rs`, `bin/role.rs`), Cargo build scripts, and test modules (`*_tests.rs` / `tests/*.rs`) — these do not need orientation headers.

Re-verify with a current-tree grep over `crates/` only; the old root `src/` tree no longer exists.

The next orientation-doc work is `///` item-level docs on public API entry points at crate boundaries, and rustdoc examples on the stable-surface functions — but that pass waits until the crate extraction (Stage 4) stabilises the public surfaces.

## Steps

1. For each file, read the first 50 lines and the public item list.
2. Write a `//!` block at the top: purpose, scope claim, key invariants, what callers can assume.
3. Model on <RepoFile path="crates/jackin-core/src/env_model.rs">crates/jackin-core/src/env_model.rs</RepoFile> (the 3-element exemplar: purpose + scope + history).
4. Do not add `///` item-level docs in this pass — only `//!` module-level.

## What a good `//!` block says

- **Purpose:** What does this module do?
- **Scope:** What is it NOT responsible for? (crucial for AI agents to know where not to look)
- **Key invariants:** What must always be true when this module's public functions return?

## Caveats

Purely additive — no behavior change, no CI risk. Do before the behavioral specs since the spec presupposes the module has a stated contract.
