**Status**: In progress

The local-artifact cutover has landed: when configured and permitted by network policy, production processes export traces, logs, and metrics directly over OTLP/gRPC; the configured backend owns durable history, and product processes retain only bounded current-invocation state for operator UI. They do not write telemetry files, local logs, sidecars, spools, or Collector configuration.

The landed foundation includes the semantic-convention 1.43 extension registry, generated vocabulary, typed facade APIs, direct providers, bounded current-invocation progress, and substantial governed instrumentation. Complete Weaver reproducibility and schema enforcement, remaining generic call-site migration, operation and propagation coverage, event/metric families, privacy gates, and conformance evidence remain active work. Stable process Resource identity and the rule that correlation identifiers stay out of Resource and metric dimensions remain binding requirements.

`jackin diagnostics validate` emits markers, enforces a shared bounded force-flush, and requires a new exporter-success delta with no new failure for each signal. Its live-receiver acceptance test also proves failure against a stopped endpoint. Persistent host and Capsule daemons expose sanitized resolved per-signal fingerprints, typed configuration failures, exporter health, flush state, shutdown state, and Capsule direct-export coverage over their control protocols. `--debug` remains operator troubleshooting output and may raise governed telemetry verbosity when OTLP is enabled; it does not create an artifact.

## Remaining work

The implementation has removed the legacy JSONL run timeline, command-output telemetry sidecars, Capsule and host-daemon log files, local log readers/commands, file-specific telemetry configuration, legacy `jackin.*` and `parallax.run.id` telemetry namespaces, and the misleading telemetry-store name. Test-only PTY captures, render fixtures, and allocation artifacts remain explicitly outside production telemetry.

Before this roadmap can close, the live receiver must prove the validation markers and direct host/daemon/Capsule delivery, and the remaining instrumentation, conformance, and acceptance evidence must be completed. SQLite connection establishment is a bounded `db.client` operation with `db.operation.name=connect`; database paths and raw driver errors are prohibited telemetry fields. Every `Err` consumed or propagated at its semantic ownership boundary must reach OpenTelemetry exactly once through a typed, privacy-safe result helper or a more specific registered boundary; raw error values are never exported implicitly. A flush request or successful command exit alone is not treated as proof that the backend accepted all three signals.

Managed usage probes own both their bounded subprocess and external stdio RPC boundaries. The closed RPC vocabulary permits systems `codex.app-server` and `grok.acp` with methods `initialize`, `initialized`, `account/rateLimits/read`, `account/read`, and `x.ai/billing`; arguments, request parameters, response payloads, account identity, executable paths, and raw errors remain prohibited.

Profiles remain deferred until the OpenTelemetry Rust profiling signal is stable and receives a separate design. Backend deployment, storage, dashboards, alerts, authentication, and retention remain owned outside jackin❯.
