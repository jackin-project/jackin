## Purpose

This page is the consolidated list of acknowledged review findings that still need verification or follow-up. It is a catalog, not a design doc. When a finding graduates into concrete work, move it to a dedicated roadmap page; when a finding is fixed, remove it rather than keeping a stale resolved entry.

## Current catalog posture

The previous catalog carried broad claims from an older review pass and had drifted against the implementation. Do not cite those claims as current without a fresh code check. This page now keeps only the active work shape and marks evidence that must be refreshed.

## Needs fresh evidence

- **Launcher preview and mount-resolution parity.** Re-check whether the interactive launcher preview still omits selector-scoped global mounts or other access that the real launch path resolves.
- **Secret redaction in command failures.** Re-check ShellRunner and Docker/host command error formatting for raw argv or env values that can leak secrets in terminal-visible failure messages.
- **Mount validation boundaries.** Re-check scoped/global mount persistence, nested sensitive-file warnings, overlap with runtime-owned paths, and write-time rejection of invalid state.
- **Config write concurrency.** Re-check whether config persistence still uses last-writer-wins full-file rewrites without locking or merge protection.
- **Workspace role/agent selectors.** Re-check whether `allowed_roles`, `default_role`, and trust-grant selectors are parsed and validated before persistence.
- **Runtime-disable env protection.** Re-check whether host `JACKIN_DISABLE_*` passthrough vars are reserved against role-manifest override.
- **Subprocess timeout policy.** Re-check whether command execution has bounded timeouts on paths where a hang affects the operator.
- **Reproducibility and provenance.** Still open as a dedicated roadmap item: [Reproducibility and Provenance Pinning](/roadmap/reproducibility-pinning/).
