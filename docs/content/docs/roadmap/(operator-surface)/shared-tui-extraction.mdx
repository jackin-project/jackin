# Shared TUI Extraction Follow-through

**Status**: Done

TermRock is the single implementation and conformance owner for every
product-neutral terminal component used by jackin❯. jackin❯ pins one exact
TermRock Git revision (`0.11.0` / `f802fcc…`), assembles surfaces through
`Theme::default()` and semantic `Role` helpers, and keeps only product-owned
tokens (brand pill, rain/launch effects, agent menu and status accents) in
`jackin-core`. Exact component visual contracts live in TermRock tests and
catalog previews; jackin❯ tests cover product state, composition, and
interaction outcomes.

The design record that authorized the extraction has been retired (the
`reference/research/shared-tui-extraction` tree is gone). Normative TUI
guidance lives under [TUI Architecture](/reference/tui/architecture/),
[TUI Components](/reference/tui/components/), and
[TUI Visual Design](/reference/tui/visual-design/). TermRock migrations are
indexed in the [TermRock repository](https://github.com/tailrocks/termrock)
(`MIGRATING.md`).

## Goal

Make the latest TermRock `main` revision the single implementation and
conformance owner for every product-neutral terminal component used by jackin❯.
jackin❯ assembles those components directly and does not recreate their
palettes, geometry, focus behavior, scrolling, hit-testing, or rendering.

## Delivered

1. **TermRock default design system** — `Theme::default()` is the Tailrocks
   phosphor preset (single-line borders, green focused borders, gray inactive
   borders). `Role::Text` is ordinary body text; `Role::TextStrong` is bold
   headings. Shared containers expose first-class focus emphasis
   (`PanelEmphasis`, `Viewport::emphasis`).
2. **jackin❯ adoption** — workspace and fuzz lockfiles pin one TermRock
   revision. Console, launch, and Capsule use TermRock components and
   `Theme`/`Role` helpers for neutral presentation. Product palette is limited
   to brand and domain tokens.
3. **Visual conformance ownership** — TermRock tests and catalog previews own
   glyphs, role mapping, and deterministic render. jackin❯ tests assert product
   state projection and composition.
4. **Forward-only TermRock API documentation** — sequential migrations
   through `0020` (LogPane oldest navigation) and prior (`0018`–`0019` theme-explicit
   scroll / bounded LogPane scrollback, `Viewport`, `PanelEmphasis`, neutral `DialogScroll`) are indexed on TermRock `main` with
   before/after consumer guidance. TermRock work landed on `main` only (no
   TermRock PRs).

## Acceptance evidence

- [x] `Theme::default()` alone reproduces the canonical Tailrocks look and feel across every shared component, including single-line borders, green focused borders, and gray inactive borders.
- [x] TermRock semantic roles cover ordinary and strong text plus every reusable meaning previously served by duplicated jackin❯ theme constants.
- [x] Shared widgets expose the focus, emphasis, layout, scrolling, and interaction capabilities jackin❯ needs without consumer-side role substitution or copied rendering.
- [x] Generic jackin❯ rendering imports no raw shared palette constants from `jackin_core`; the remaining product palette contains only documented brand or domain semantics.
- [x] Console, launch, and Capsule use available TermRock components and best-practice APIs instead of manually painting equivalent neutral UI.
- [x] Exact component visual contracts are tested once in TermRock; jackin❯ tests concentrate on product state, composition, and outcomes rather than shared glyph or RGB implementation details.
- [x] Every breaking TermRock change has an indexed sequential migration file with before-and-after consumer guidance.
- [x] jackin❯ pins one exact latest reviewed TermRock revision everywhere, including every fuzz lockfile, and its TUI-related Rust gates pass against that revision.
- [x] TermRock format, Clippy, catalog, and deterministic-render gates pass on the consumed revision (full TermRock CI matrix runs on TermRock `main`).

## Not part of this item

- migrating or validating another Tailrocks product;
- crates.io publication, Windows support, terminal degradation modes, or a shared testing crate;
- moving the literal jackin❯ brand, product state, product wording, effects, Tokio runtime policy, secrets, host-open policy, or raw terminal ownership into TermRock;
- preserving obsolete APIs or visual behavior solely for backward compatibility.
