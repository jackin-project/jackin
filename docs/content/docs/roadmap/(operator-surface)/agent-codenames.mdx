---
title: "Agent codenames for parallel session coordination"
---

**Status**: Open — design proposal; source audit found no shipped `JACKIN_AGENT_CODENAME`, `AgentRecord`, codename registry, or `jackin-capsule agents` implementation yet.

## Problem

jackin❯ operators routinely open multiple tabs inside one Capsule session. Each tab can run the same agent, provider, workdir, and branch. Today those tabs are indistinguishable from inside the container: they share `JACKIN_AGENT`, `JACKIN_INSTANCE_ID`, and other runtime env. That anonymity makes self-organized coordination hard. Agents cannot say “I am badger, I own this task” in a stable way, and a restarted agent process cannot recover a tab-local identity after context loss.

## Proposed capability

Provide two infrastructure-owned identity surfaces:

1. **Per-tab codename.** Assign a unique, human-readable codename when a tab is created and inject it into every process in that tab as `JACKIN_AGENT_CODENAME`. The codename is a tab property, not a process property, so replacement agent processes inherit the same identity.
2. **Read-only agent registry.** Add `jackin-capsule agents` to query the Capsule daemon for active/exited tab records: codename, agent runtime, provider, start time, exit time, and status. Agents can read the registry; only Capsule writes it.

jackin❯ should not prescribe a coordination format or task-tracking protocol in this item. The infrastructure supplies stable identity and a trusted registry; operators and role authors decide how agents use it.

## Implementation plan

### Phase 1 — Codename infrastructure and registry command

- Embed a curated wordlist in `jackin-capsule`.
- Add a `codename: String` field to tab/session state.
- Track live and retired codenames so names are never reused within a container lifetime.
- Maintain an append-only `AgentRecord` history in the Capsule daemon.
- Inject `JACKIN_AGENT_CODENAME` in the agent command environment.
- Reserve `JACKIN_AGENT_CODENAME` against role-manifest/env override.
- Add `jackin-capsule agents` and `jackin-capsule agents --format json`.
- Render a tab-hover codename tooltip in Capsule without changing the resting tab-strip label.

### Phase 2 — Operator and role-author documentation

- Add an operator/role-author guide explaining `JACKIN_AGENT_CODENAME`, `jackin-capsule agents`, JSON output, and practical parallel-session coordination patterns.

### Phase 3 — Integration after feedback

- Persist the registry across container restarts only if operators need post-restart history.
- Surface codenames in host `jackin status` when a running Capsule registry is cheap to query.
- Consider configurable wordlists or operator-assigned codenames only after the generated identity model proves useful.
