**Status**: Open — diagnostics JSONL, reveal, and open surfaces exist today but are scheduled for retirement by [Unified OpenTelemetry observability](/roadmap/unified-otel-observability/); project join and a non-telemetry redacted debug bundle remain unimplemented

## Shipped facts

jackin❯ currently writes diagnostics JSONL for runs and exposes reveal/open affordances for those host-side diagnostic artifacts from launch failure and Debug info surfaces. That is shipped behavior, not the future bundle substrate: the unified observability cutover makes production telemetry OTLP-only and removes local telemetry artifacts, readers, and reveal actions. A shareable debug bundle therefore covers redacted product/configuration state while telemetry retention and queries remain the operator's backend responsibility.

## Remaining work

### 1. Redacted debug bundle

Add a diagnostic export command, name still open (`jackin debug export` or similar), that packages a local `.tar.zst` bundle containing redacted workspace config, redacted role manifest, redacted env/auth summary, jackin❯ and Capsule versions, OS info, container/runtime state, and exporter configuration/health with endpoint credentials removed. An optional run id may be recorded as a backend correlation reference, but the bundle never queries the backend or contains logs, traces, metrics, OTLP payloads, or another telemetry serialization.

Redaction must hide token/API-key values and 1Password vault/item/field specifics while preserving enough structure to debug resolution failures. Bundles stay local; no upload service and no import/replay command in this item.

### 2. `jackin join` research

Research project-level onboarding separately from debug export. `jackin join <org>/<project>` would let a repo publish jackin❯ workspace setup information, then guide a new contributor through trust, workspace choice, credentials, clone/setup, and first launch. Open questions remain: config file shape, private-repo auth, trust model, multi-workspace monorepos, and whether prerequisites are installed or only diagnosed.

## Non-goals

- Do not implement `jackin join` before the project-config trust model is designed.
- Do not preserve diagnostics JSONL, create an OTLP file export, or query a configured backend as part of the bundle.
- Do not store bundles remotely.
