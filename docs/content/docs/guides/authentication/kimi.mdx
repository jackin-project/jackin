[Kimi](https://www.kimi.com) provides an Anthropic-compatible coding endpoint (`https://api.kimi.com/coding/`) backed by the `kimi-for-coding` model. Because the endpoint speaks the same protocol as Anthropic's API, Claude Code works against it without modification â€” jackin' injects the redirect at session-spawn time.

When `KIMI_CODE_API_KEY` is configured, opening a new Claude tab presents a provider picker with **Kimi** as one of the options. Operators without a Kimi key see no change: Kimi does not appear in the picker.

<Aside type="note">
This guide covers using Kimi's API as a **provider for Claude Code** sessions â€” routing Claude Code to Kimi's endpoint. It is separate from the [Kimi Code](/guides/authentication/agents/) agent runtime, which runs the Kimi Code CLI itself as an agent inside the container. Both can be active in the same jackin' installation.
</Aside>

<Aside type="tip">
The Kimi provider is configured in the **Auth** tab of `jackin console` under the **Kimi** row. Configure it the same way as Z.AI or MiniMax â€” not by hand-editing any config file.
</Aside>

## Subscribe and get an API key

1. Open [Kimi membership pricing](https://www.kimi.com/membership/pricing), sign in, and purchase a Kimi membership. The coding endpoint benefit is included in paid membership plans.
2. After your membership is active, open the [Kimi Code Console](https://www.kimi.com/code/console) and go to **API Keys**. Create a new key â€” the key is shown only once and starts with `sk-ki`. Copy it immediately.
3. You can create up to 5 keys. Keep your key private. Store it in jackin' through the Auth settings below; do not paste it into Claude Code's host settings.

<Aside type="note">
Kimi's coding endpoint (`api.kimi.com/coding`) is distinct from the general Moonshot platform endpoint (`api.moonshot.ai/v1`). The membership benefit is the coding endpoint; the key is managed in the Kimi Code Console, not in the Moonshot platform. Kimi's coding subscriptions also carry weekly and rate-limit windows with no carryover â€” plan your usage accordingly.
</Aside>

## Configure your Kimi API key in jackin'

1. Open `jackin console`, select your workspace, and switch to the **Auth** tab.
2. Scroll to the **Kimi** row and press `Enter` or `Space` to open the auth form. (This row is for the Kimi provider credential. The **Kimi Code** row controls the Kimi Code CLI agent runtime.)
3. Set the mode to **API Key** (press `Space` to cycle).
4. Enter your `KIMI_CODE_API_KEY` â€” either a literal key or a 1Password reference resolved at launch time.
5. Save the form.

The key is written to the `[env]` map in your config file at the scope you chose (global or per-workspace). A workspace-level key overrides the global one.

To remove Kimi, open the same row, cycle the mode back to **Ignore**, and save.

## Using the provider picker

With `KIMI_CODE_API_KEY` configured:

**At workspace launch** â€” launch a workspace and choose the Claude agent. Before the container starts, a provider picker appears. Use arrow keys and `Enter` to pick. `Escape` cancels back to the agent picker.

**In the multiplexer** â€” press `N` inside a running container to open a new agent tab. When you pick Claude, the same provider picker appears if multiple providers are available.

## Tab labels

The tab strip label reflects the active provider:

| Active provider | Tab label |
|---|---|
| Anthropic | `Claude (Anthropic)` |
| Kimi | `Claude (Kimi)` |

When only Anthropic is configured, the bare `Claude` label is used â€” no suffix.

## Scopes

`KIMI_CODE_API_KEY` follows the standard 4-layer env resolution â€” most-specific wins:

**per-(workspace Ă— role) > per-workspace > global > absent**

Set a global key to have Kimi available in every workspace, or set a per-workspace key to restrict it to specific projects.

## How the redirect works

When you select Kimi in the provider picker, jackin' injects environment variables into the spawned Claude Code process at launch time â€” not into any shared config file:

- `ANTHROPIC_AUTH_TOKEN` â€” set to your `KIMI_CODE_API_KEY` value
- `ANTHROPIC_BASE_URL` â€” set to `https://api.kimi.com/coding/`
- `ANTHROPIC_DEFAULT_OPUS_MODEL` â€” set to `kimi-for-coding`
- `ANTHROPIC_DEFAULT_SONNET_MODEL` â€” set to `kimi-for-coding`
- `ANTHROPIC_DEFAULT_HAIKU_MODEL` â€” set to `kimi-for-coding`
- `API_TIMEOUT_MS` â€” set to `3000000` (50-minute timeout for long-running operations)
- `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` â€” set to `1`

`kimi-for-coding` is the single model served by the coding endpoint, so all three Claude tier slots map to the same model. The timeout prevents premature disconnection on long agent runs through the proxy.

**No other tabs or sessions in the container are affected.** Anthropic tabs continue to use the container's own credentials.

Your `KIMI_CODE_API_KEY` is never written to the logs, including `--debug` output.
