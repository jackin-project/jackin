# Conventional Commits Adoption + Franchise-Vocabulary Scrub Completion

This design covers two coupled cleanup workstreams across the eight repos under
`/Users/donbeave/Projects/jackin-project/`:

1. **Adopt Conventional Commits as the documented and enforced commit-message
   convention**, with backfill of historical non-conforming commits.
2. **Finish the franchise-vocabulary scrub** that commit `ff2a859` ("Drop
   franchise-specific vocabulary across the repo") started but did not
   complete in `jackin`, and extend the same scrub to peer repos that still
   contain phosphor-themed prose or constants.

The two workstreams are coupled because the cleanest way to remove franchise
references from historical commit messages is the same `git filter-repo` pass
that backfills Conventional Commits.

## Goals

- Every repo's `AGENTS.md` documents Conventional Commits as the required
  format for new commits, with a small allowed-types list and a link to the
  spec.
- New commits going forward across all eight repos conform to Conventional
  Commits.
- Historical `main`-branch commits in repos where rewrite is cheap and useful
  are reworded to conform; repos where rewrite would damage automation
  traceability keep their history intact.
- The working tree of every repo is free of Matrix-franchise marketing prose
  and free-form franchise references in code identifiers.
- Public-facing surfaces (Cargo description, Homebrew formula `desc`, README
  taglines, AGENTS.md prose, landing copy, formula descs, agent personality
  prose) contain no franchise vocabulary.
- The `jackin` app continues to build, test, and run with no regressions
  after the rename of `PHOSPHOR_GREEN/DIM/DARK` color constants.

## Non-Goals

- Renaming the `jackin-agent-smith` or `jackin-the-architect` repos. These
  identifiers are pinned to public GitHub URLs and renaming is a breaking
  change. The previous commit `ff2a859` deliberately deferred this; this
  design honors that boundary.
- Renaming `docker/construct/` directory or the `jackin-construct` Docker
  image. Same rationale — public/automation-pinned identifiers.
- Rewriting the 91 release-bot commits in `homebrew-tap`
  (`jackin@preview 0.6.0-preview.412+55a18ea` pattern). These are
  machine-generated by upstream CI; rewriting breaks the SHA cross-reference
  back to `jackin` build SHAs and does not prevent future commits from
  looking identical.
- Squashing or flattening GitHub merge commits. They will be reworded
  (`chore: merge pull request #N (<branch>)`) but the merge graph is
  preserved.
- Adding commit-message linting to CI. That's a follow-up that should be its
  own design once the convention is documented; this design only documents
  and backfills.

## Chosen Approach: Two Phases, Forward-Only First

### Phase 1: Forward-only fixes (non-destructive)

Per repo, on a feature branch, opened as a PR per `AGENTS.md` line 11
("Never commit directly to `main`"):

1. **Add a `## Commit Messages` section to AGENTS.md** documenting:
   - All commits MUST follow [Conventional Commits 1.0.0](https://www.conventionalcommits.org/en/v1.0.0/).
   - Allowed types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`,
     `test`, `build`, `ci`, `chore`, `revert`.
   - Scope is optional but encouraged (e.g., `feat(launch): ...`).
   - Breaking changes use `!` after type/scope (e.g., `feat!: ...`) or a
     `BREAKING CHANGE:` footer.
   - Squash-merging PRs preserves the PR title as the commit subject, so PR
     titles must also conform.
2. **Finish the franchise-vocabulary scrub** in working tree only:
   - **`jackin`**: replace `// PHOSPHOR_GREEN`, `// PHOSPHOR_DIM`, `// PHOSPHOR_DARK`
     comment labels in `docs/src/components/landing/rainEngine.ts` lines
     10/12/13 with `// PHOSPHOR_BRIGHT`, `// PHOSPHOR_MID`,
     `// PHOSPHOR_DEEP` (matching the naming the previous commit established
     for the Rust-side constants). Update the test name on
     `docs/src/components/landing/rainEngine.test.ts:14` to
     `ageToColor returns phosphor bright for age 3-5`. Grep for any
     other `MATRIX_` references and update.
   - **`homebrew-tap`**: edit the `desc` line in both `Formula/jackin.rb`
     and `Formula/jackin-preview.rb` to a neutral phrasing, e.g.,
     `"CLI for orchestrating AI coding agents in Docker containers at scale"`.
   - **`jackin-agent-smith`**: rewrite AGENTS.md/CLAUDE.md prose. Replace
     "Agent Smith personality" framing with neutral functional description:
     `"A Claude Code agent image extending projectjackin/construct:trixie
     with the code-review and feature-dev plugins pre-configured for
     code-review-focused work."` Repo-name identifier (`jackin-agent-smith`
     in URLs) stays.
   - **`jackin-the-architect`**: remove the `> "I am the Architect. I
     created the Matrix."` quote from README.md line 5. Soften the
     AGENTS.md line 3 rationale to: `Named "the-architect" because it has
     the broadest operator capability of the agent images — it can
     manage the entire jackin-project repo collection.`
   - Other repos: no working-tree franchise refs found, no scrub needed.
3. **Verify each repo's tooling passes**:
   - `jackin`: `cargo fmt -- --check && cargo clippy && cargo nextest run`
     plus `cd docs && bun run build` (per repo `AGENTS.md` line 36).
   - `jackin-the-architect`, `jackin-agent-smith`: `docker build` smoke if
     Dockerfile changed (it doesn't here — only docs).
   - `homebrew-tap`: `brew audit --strict --formula Formula/jackin.rb`
     and same for `jackin-preview.rb` if `brew` is available; otherwise
     visual diff review.
   - `jackin-github-terraform`: `tofu fmt -check && tofu validate`.
4. **Open one PR per affected repo**, squash-merge with a Conventional
   Commits subject:
   - `jackin`: `docs: complete franchise-vocabulary scrub + adopt Conventional Commits` (one PR doing both since they're coupled — small AGENTS.md edit + small rainEngine rename)
   - `homebrew-tap`: `chore: scrub franchise prose from formula descs + adopt Conventional Commits`
   - `jackin-agent-smith`: `docs: neutralize agent personality framing + adopt Conventional Commits`
   - `jackin-the-architect`: `docs: remove Matrix quote from README + adopt Conventional Commits`
   - Other repos (jackin-dev, jackin-marketplace, jackin-github-terraform,
     validate-agent-action): one tiny PR each, `docs: adopt Conventional Commits`.

Phase 1 is fully reversible. Sibling agents on feature branches in
`homebrew-tap` and `jackin-github-terraform` are unaffected.

### Phase 2: History rewrite (destructive, gated per repo)

Sequenced **after Phase 1 PRs land**, and **after the user confirms sibling
agents have paused** (or accepts that they will need to rebase their feature
branches against rewritten `main`).

For each repo where rewrite is decided to be worthwhile (see decision
matrix below):

1. **Build a `git filter-repo --message-callback` Python script** mapping
   every non-conforming commit subject to a Conv-Commits subject. Mappings
   are deterministic:
   - `Merge pull request #N from <owner>/<branch>` →
     `chore: merge pull request #N (<branch>)`
   - `release: vX.Y.Z` → `chore(release): vX.Y.Z`
   - Initial commits → `chore: initial commit`
   - Free-form messages: hand-curated mapping table reviewed by user before
     execution. Each entry: `<old_subject> → <new_subject>`.
2. **Run on a fresh clone first** to verify the result. Diff `git log
   --oneline` before and after. Verify count of conforming commits
   transitions from current% to 100%.
3. **Show user the proposed `git log --oneline -50` of the rewritten
   history**, get explicit per-repo go-ahead.
4. **`git push --force-with-lease origin main`** with the user's lease
   (so any drift since the audit aborts the push safely).
5. **Delete the local pre-rewrite refs** only after user confirms remote
   looks correct via `gh repo view <repo> --web`.

## Decision Log

### Decision 1: Skip rewriting `homebrew-tap` release-bot commits
**Decision:** Yes. The 91 commits matching `jackin@preview <ver>+<sha>`
embed a SHA correlation back to the upstream `jackin` build that produced
them. Rewriting them changes their SHAs but does not change the embedded
`+<sha>` reference, creating a permanent mismatch. Future bot commits will
also continue to use the same format until the bot itself is changed
(out of scope). Net: pure loss.

### Decision 2: Reword GitHub merge commits, don't flatten
**Decision:** Reword. `Merge pull request #N from owner/branch` becomes
`chore: merge pull request #N (branch)`. Flattening (squash-rebase) would
satisfy the linter but also destroys the merge-graph topology, which is
useful for understanding how parallel branches integrated.

### Decision 3: Honor identifier-vs-prose boundary set by `ff2a859`
**Decision:** Honor it. Repo names (`jackin-agent-smith`, `jackin-the-architect`),
directory names (`docker/construct/`), Docker image names
(`jackin-construct`), and Terraform variable values referring to the same
are kept. Only marketing prose, README quotes, AGENTS.md personality
framing, formula `desc` lines, and code constants whose names directly
quote the franchise (`PHOSPHOR_GREEN` etc.) are scrubbed. This matches the
boundary the previous commit's body explicitly set.

## Risk Model

| Risk | Likelihood | Mitigation |
|---|---|---|
| Force-push to `main` overwrites in-flight PR base on a sibling agent's branch | Medium (2 sibling agents currently on feature branches per audit) | User confirms agents paused before Phase 2; Phase 1 first gives agents time to merge their work. |
| `git filter-repo` mangles a commit due to typo in mapping table | Low | Run on fresh clone first, diff result, user reviews 50-line log before push. |
| Renaming MATRIX_* comment labels accidentally affects runtime | Very low — they're comments, not identifiers | The rgb() values inline don't change; `bun test` and `bun run build` after edit confirm no regressions. |
| Public commit URLs in external docs/issues break after rewrite | High for cited SHAs | Accept per user explicit authorization; mitigate by tagging `pre-rewrite-2026-04-21` in each repo before Phase 2 so old SHAs remain reachable via tag. |
| Brew formula edits break `brew install jackin` | Low (only `desc` field changes; no `url`/`sha256` change) | `brew audit --strict` if available; visual diff otherwise. |
| `cargo nextest run` fails due to test file referencing renamed constant | Medium | Audit-driven — `rainEngine.test.ts` is named in advance, gets the rename in the same diff. |
