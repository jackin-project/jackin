03 · The Problem

Unrestricted agents need unrestricted containment.

Agents like Claude Code are most productive when they run without permission prompts. But on your host, unrestricted means everything is visible — including things you never meant to share.

  • ✕Every file. SSH keys in ~/.ssh, browser cookies, your password manager's unlocked session.
  • ✕Every command. rm -rf, curl to external endpoints, arbitrary system modifications.
  • ✕Every package. Any dependency the model decides to install — supply chain and all.
  • ✕Every credential. AWS, Docker Hub, GitHub tokens — anything in your shell environment.
  • ✕Every mistake. A single wrong command is a bad day, not a rollback.
~ — unrestricted agent
$ ls -la ~ drwx------ .ssh/ # private keys -rw------- .aws/credentials # long-lived tokens -rw------- .env # production secrets drwxr-xr-x .config/ drwxr-xr-x Projects/ ... all visible to the agent.
"Running an unrestricted AI agent on your host is like inviting a brilliant contractor into your home and saying 'do whatever you want.'" — docs / getting-started / why

Agent classes are GitHub repos (owner/agent). jackin's own built-ins (agent-smith, the-architect) can be loaded by short name; every third-party agent uses the full owner/agent form. Switch orgs to see how a single operator spans multiple projects without mixing tooling.
Workspaces bundle a workdir and its mounts — each mount declares its source, destination, and whether it's read-write or read-only. The access boundary is explicit, not implied. (Switch to @chainargos and try frontend-engineer × docs-only to see access rejection in action.)
05 · How it Works

Four commands. One loop.

jackin' is designed around a small, memorable vocabulary. Most of daily use is just these four.

01
jackin load agent-smith
Clone the agent repo, build the derived image, create a per-agent network, launch DinD, start Claude Code inside. You're instantly in.
→ Pulling construct:trixie... OK → Building derived image... OK → Container started. Agent loaded.
02
jackin hardline agent-smith
Closed your terminal but the agent is still running? Reattach to the existing container session with one command.
→ Reattaching to jackin-agent-smith... OK → Session restored. History intact.
03
jackin load the-architect monorepo
Load a second agent — different tool profile, same or different workspace. They run in parallel, isolated from each other.
→ Two agents active: agent-smith, the-architect → Separate networks, separate state.
04
jackin eject agent-smith
Stop one agent cleanly — state persists for next time. Or jackin exile to pull everyone out at once.
→ Stopping agent-smith... OK → State saved. Network removed.
06 · Security Model (honest version)

Strong where it counts. Honest about the rest.

jackin' is container isolation, not hypervisor isolation. We document exactly what that means. No "enterprise-grade" hand-waving.

Isolated

  • FilesystemAgent sees only mounted dirs. Host home, SSH keys, credentials are out of reach.
  • NetworkPer-agent Docker network. No access to host services unless explicit.
  • Docker daemonEach agent has its own DinD sidecar. Can't touch your host's daemon.
  • Shell environmentAgent doesn't inherit your env vars. Tokens stay on the host.
  • Installed packagesAll installs happen inside the container. Host stays clean.

Shared (be aware)

  • KernelContainer, not microVM. A kernel exploit crosses the boundary.
  • CPU / memoryNot hard-partitioned. Resource pressure affects host.
  • Mounted pathsWhatever you mount, the agent can read (and write, unless :ro).
  • Trust in agent sourceAgent repo authors can define what runs in the image.

Want microVM-level isolation? Docker Sandboxes is stronger there. Want open, local-first orchestration with role-specific agents and workspaces as code? That's where jackin' is built to live.

07 · Ecosystem

Agents are git repos. Build your own.

An agent class is a Dockerfile plus a small manifest. Clone, version, PR-review. Share across an org.

AS

agent-smith

github.com/jackin-project/jackin-agent-smith

Default general-purpose agent. Starts here and most first-time operators never need another. Loadable by short name because it's a jackin' built-in.

TA

the-architect

github.com/jackin-project/jackin-the-architect

Full Rust toolchain, ripgrep, just — the agent used to develop jackin' itself. Also loadable by short name.

+

your-org/your-agent

github.com/your-org/jackin-your-agent

Any third-party agent is loaded by its full owner/agent name. The GitHub repo always uses the jackin- prefix. Write a Dockerfile that extends the construct, add a manifest, push it up.
